# Checklists

Pre-flight / post-flight checklists for common operations land in this directory.

Each checklist is a markdown file with concrete tickable items, ordered by when they should fire (before, during, after the operation). Empty today — populated as patterns surface that benefit from a fixed checklist over loose prose.

Examples of what would belong here:
- `release-checklist.md` — steps to bump a plugin version, update marketplace.json, push, tag, and verify install.
- `new-plugin-checklist.md` — boilerplate steps to scaffold a new domain plugin.
- `incident-postmortem-checklist.md` — what to capture after an issue.
