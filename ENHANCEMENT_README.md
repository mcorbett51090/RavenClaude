# Enhanced RavenClaude — With Learnings from agentic-harness

This directory contains **proposed enhancements** to RavenClaude that integrate key patterns from the excellent `agentic-harness` project (https://github.com/codejunkie99/agentic-harness) to produce even higher-quality, more reliable "ideal outputs" from the multi-agent team.

## What's Included

- `plugins/ravenclaude-core/CLAUDE.md` — **Enhanced core constitution** with new sections on:
  - Structured Output Protocol (JSON + delimiters for reliable parsing)
  - Focused Task Execution
  - Run Artifacts standard (`.ravenclaude/runs/<id>/`)
  - Context Hygiene
  - Integrated with existing hierarchical dispatch, Researcher, and Grounding Protocol.

- `plugins/ravenclaude-core/skills/structured-output.md` — New reusable skill for applying the output protocol.

- `plugins/ravenclaude-core/templates/run-artifacts/` — Templates for standardized artifacts:
  - `summary.md.template`
  - `structured-result.json.template`

- `docs/ENHANCEMENT_PROPOSAL.md` — Full rationale, benefits, and integration instructions.

## Quick Start to Apply

1. Copy `plugins/ravenclaude-core/CLAUDE.md` into your RavenClaude checkout (replace the existing one in the plugin).
2. Add the new `skills/structured-output.md` and the `templates/run-artifacts/` folder.
3. Update your agent prompts and rules to reference the new protocols (especially "use Structured Output Protocol for handoffs and deliverables").
4. Optionally extend hooks to encourage or validate artifact generation.
5. Test with a real workflow and inspect the new `.ravenclaude/runs/` artifacts.

These changes are **backward-compatible** and layer on top of RavenClaude's existing strengths. They directly address making outputs more parseable, focused, observable, and consistently ideal.

## Why This Matters

By combining RavenClaude's powerful multi-agent team orchestration with harness-style rigor around **structured outputs**, **task focus**, and **rich artifacts**, your agents will produce results that are:
- More reliable in handoffs
- Easier to inspect and debug
- Better suited for automation and continuous improvement
- Closer to true "ideal" production-grade engineering outputs

For full details, read `docs/ENHANCEMENT_PROPOSAL.md`.
