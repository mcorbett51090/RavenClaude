Process Flow

How the Command Center works

The dashboard turns six scattered systems into one morning loop. These diagrams trace the flow end-to-end — what the PSM does, where the numbers come from, how the priority score is computed, and how the pieces depend on each other. Pair this with the Build Plan ↗.

One sentence

Data from every system lands in a single data.json keyed by account_uid; the dashboard reads that file, scores every partner, and shows the PSM exactly who to call, why, and when — newest answer every morning.

Why this matters for the build

Because the dashboard reads only data.json, you build it against synthetic data first (the red path) and swap in real data later (the source path) without changing a line of dashboard.html.

3 · How the priority score decides who to call

Nine signals, each weighted, combine into one 0–100 score. The score is never shown without the signal that drove it.

    S6[ticket volume]
    S7[ARR]

  end

  S1 -- ×25 --> W((weighted
sum ÷ 100))

  S5 -- ×10 --> W
  S6 -- ×5 --> W
  S7 -- ×5 --> W

0-100]
  PS --> ROW[Action Center row
score · top signal · reason]
  ROW --> ACT[recommended_action

  classDef sig fill:#eef0f3,stroke:#d2d7dd,color:#1c1f23;

Tunable, not hardcoded

The weights live in priority_weights in data.json. Re-tune them after a couple weeks of use — no code change. Every row carries its top contributing signal (e.g. renewal_timing 38%) so the score is explainable, not a black box.

6 · The refresh lifecycle

How a fresh answer reaches the dashboard each day — synthetic now, real later, same shape. 

sequenceDiagram
  autonumber
  participant PSM as PSM
  participant Gen as synthesize.py / export
  participant DJ as data.json
  participant Dash as dashboard.html
  PSM->>Gen: run generator (or real-data export)
  Gen->>DJ: write fresh data.json (seeded · FERPA-safe)
  PSM->>Dash: open in browser
  Dash->>DJ: fetch ./data/data.json
  DJ-->>Dash: partners · scores · events
  Dash->>Dash: sort by priority · color by band
  Dash-->>PSM: "call these, here's why, here's when"

Same loop, two data sources

Today: python3 data/synthesize.py → open dashboard.html. Later: export real partner data into the same data/data.json → open dashboard.html. The dashboard can't tell the difference, and neither does the morning routine.
