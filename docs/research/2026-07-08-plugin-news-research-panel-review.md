## Why only 3 of 81 were implemented

The panels rank findings by whether they *could* improve a plugin. Before editing, each P0 ("plugin currently states something now-false") was checked against the **actual plugin files** — and **7 of the 10 P0s did not survive that check**:

| id | plugin | why it was NOT a correctness fix |
|---|---|---|
| 26 | staffing-operations | plugin never states the mandate — no false claim to fix |
| 50 | renewable-energy | plugin already OBBBA-aware ("post-2025 ITC shift", dated) — not a false statement |
| 52 | ecommerce-dtc | plugin never states the old $800 de minimis rule — nothing to correct |
| 53 | cannabis-operations | HIGH-BLAST: cannabis Schedule III rescheduling — momentous, hard to independently confirm; deferred to human review, not auto-merged |
| 58 | senior-care-operations | plugin ALREADY states the rule is rescinded (eff. 2026-02-02) — already correct |
| 66 | behavioral-health-practice | plugin never cites CoCM codes 99492-99494 — nothing to correct |
| 72 | automotive-dealership | plugin never references the FTC CARS Rule — nothing to correct |

This is the repo's accuracy-first / anti-churn discipline working as intended: a panel inferring "the plugin probably says X" from a description is not evidence the plugin *does*. Auto-applying 78 web-sourced edits across ~70 plugins (each a version bump + prose change) would be exactly the churn the constitution warns against, and would risk encoding claims that later shift (e.g. the Colorado AI Act date the review panel already caught as now-false). The remaining P1/P2 items are recorded below as a **reviewed backlog** for human triage rather than silently applied.
