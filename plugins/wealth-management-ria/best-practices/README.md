# Wealth-management-RIA best-practices

Atomic, enforceable rules the wealth-management-ria agents apply. Each file is one rule with a short rationale; the agents cite them by filename. Canonical decision logic lives in [`../knowledge/wealth-management-ria-decision-trees.md`](../knowledge/wealth-management-ria-decision-trees.md); these rules are the always-on priors.

| Rule | Gist |
|---|---|
| not-personalized-investment-advice | Every output is education/operational support, never a buy/sell recommendation |
| goals-before-products | Start from what the client wants to achieve, never from a product |
| surface-every-assumption | Inflation/return/longevity surfaced as a named, editable list |
| the-ips-is-the-governing-document | Allocation and trades flow from a written IPS or they're ad-hoc |
| rebalancing-is-a-written-rule | Calendar or threshold, documented — not a discretionary call |
| asset-allocation-dominates-selection | The diversification decision is the big lever; selection is downstream of it |
| after-tax-return-is-the-return | Asset location + tax-loss harvesting; mind the wash-sale rule |
| fiduciary-is-not-reg-bi | The RIA fiduciary duty and Reg BI are different standards — never conflated |
| disclose-and-manage-conflicts | A fiduciary discloses and manages conflicts; "we have none" is a disclosure failure |
| document-the-suitability-basis | KYC gathered then refreshed; the basis for advice is written and retained |
| confirm-the-client-specific-facts | Name the facts that change the answer; route them to the licensed human first |
| behavior-beats-theory | The right plan is the one the client can actually hold through a drawdown |
