# A POC needs signed, measurable exit criteria

**Status:** Absolute rule.

**Use when:** a prospect asks for a proof-of-concept or pilot.

## The rule

**No POC starts without 3-6 measurable success criteria — each with a pass/fail test — signed by the champion before kickoff.** A POC you can't fail is a POC you can't win.

## Why

An open-ended POC ("let them play with it and see") never closes: there's no agreed definition of "yes," so a strong result still leaves the buyer room to invent a new objection, and a weak one leaves you no honest verdict. The criteria are the contract; the signature is the forcing function that stops the goalposts moving the moment results arrive. A POC is the most expensive sales asset you own — spend it only when it's bounded.

## How

1. Gate first (the four preconditions: qualified pain, a champion, explicit decision criteria, reachable success). If any is missing, do a tailored demo or reference call instead.
2. Write measurable, testable, owned criteria tied to discovered pain (the POC success-criteria template).
3. Add exit/kill rules and a single extend rule; time-box and scope (explicit out-of-scope).
4. Get the champion's signature before kickoff.

## Exceptions

- **A self-serve free trial / sandbox** is not a POC and doesn't need this — it's a marketing-led motion with no SE commitment. The rule applies the moment an SE is committing time to prove something for a specific deal.
