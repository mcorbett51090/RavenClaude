# Separate deploy from release from experiment from launch

Shipping code to production, exposing it to users, measuring it against a control, and turning it on for everyone are four distinct, independently reversible events — and conflating them is how a routine deploy becomes an outage. Flags are what decouple them: deploy dark so the code is in production but inert, release progressively to a slice, run the experiment on a randomized fraction with exposure logging, then launch by flipping the flag for all. Each step has its own gate and its own rollback. A team that releases by deploying has tied its blast radius to its deploy cadence and given itself no way to turn a bad change off without shipping another. The reversibility is the entire point; preserve it by keeping the four events separate.
