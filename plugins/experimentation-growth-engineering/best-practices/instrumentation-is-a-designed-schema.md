# Instrumentation is a designed schema, not ad-hoc events

Define a tracking plan — events, properties, naming conventions, and types — before instrumenting, and validate events against it as code. Ad-hoc events with inconsistent names and untyped properties are the root cause of nearly every 'our analytics data is a mess' complaint: garbage events in means no trustworthy analysis out. The tracking plan is the schema that makes the behavioral data usable.
