# Make assignment deterministic and prefer server-side evaluation

A user must land in the same variant every time they're evaluated, so assignment is a pure hash of a stable id plus the experiment key — never a coin flip on each request and never tied to a churning anonymous cookie — or the same person flickers between variants and contaminates both arms. Prefer evaluating that hash server-side: client-side flag evaluation leaks the variant to anyone reading the bundle, flashes the wrong UI before the flag resolves, and is trivially defeated by ad blockers and bot traffic. Where a client-side decision is unavoidable, bootstrap it from a server-rendered assignment so it can't flicker. Sticky, deterministic, server-evaluated assignment is the floor beneath every trustworthy result; get it wrong and SRM and dilution follow.
