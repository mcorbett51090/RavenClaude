# The cloud is the layer above — define the contract, hand off the build

This plugin runs on the device and emits telemetry; the ingest pipeline, the backend, and the companion app belong to `data-streaming-engineering` / the cloud plugins / `mobile-engineering`. The device team's job at the seam is to define the contract precisely — the telemetry topics/endpoints, the payload schema and units, the cadence and QoS, the provisioning/control surface (BLE/Wi-Fi) — and then hand the build upward, not to grow a cloud ingest pipeline or an app inside the firmware repo. A crisp contract (versioned payloads, explicit units, a documented store-and-forward behaviour on a lossy link) lets the cloud team build against a stable target while the device team owns the metal. Doing cloud ingest or app work here is scope creep that blurs the ownership boundary and ships two half-built layers instead of one solid device contract.
