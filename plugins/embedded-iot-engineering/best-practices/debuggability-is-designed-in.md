# Debuggability is designed in, not bolted on

A field device you cannot diagnose is a field device you cannot fix. Design the diagnostic path up front: keep SWD/JTAG access reachable (or a documented re-enable path if it's locked for security), write a fault handler that captures the register context, stacked frame, and fault status into retained/no-init RAM so it survives the reset, and ship structured logging that persists across a reboot (a ring in retained RAM, RTT, or a small flash log region). Record and surface the reset cause every boot — power-on vs. watchdog vs. brown-out vs. fault — so a field reboot tells you why. Keep a low-overhead trace path (ITM/SWO or RTT) that won't perturb real-time timing the way `printf`-over-UART does. The cost of one retained-RAM crash dump is trivial against a fleet of bricks you can only guess about.
