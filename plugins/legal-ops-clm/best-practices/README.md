# Legal-ops-clm best-practices

Atomic, enforceable rules the legal-ops-clm agents apply. Each file is one rule with a short rationale; the agents cite them by filename. Canonical decision logic lives in [`../knowledge/legal-ops-clm-decision-trees.md`](../knowledge/legal-ops-clm-decision-trees.md); these rules are the always-on priors.

| Rule | Gist |
|---|---|
| operational-support-not-legal-advice | Process support only; a human lawyer owns every legal-judgement call |
| structured-intake-not-a-dm | One structured front door; triage by risk and value, not arrival order |
| the-playbook-needs-a-bright-line-escalation-trigger | Self-serve the easy majority; write the "get a lawyer" line as a bright line |
| standard-fallback-walk-away-decided-once | Each key clause: preferred / acceptable / never-acceptable, set by a lawyer |
| flag-material-deviations-not-every-comma | Surface what changes risk; every flag carries a tier and a named approver |
| signature-is-the-start-not-the-end | Each obligation becomes a tracked item with an owner and a trigger |
| track-the-notice-window-not-just-expiry | Auto-renew fires unless you give notice in the window; alert in tiers |
| every-metric-pairs-with-a-decision | Cycle time / backlog / renewal-at-risk tied to an action, never a vanity count |
