# Triangulate a valuation

You are running `/corporate-development-ma:triangulate-a-valuation` for `$ARGUMENTS`. Run it the way the team's specialists would — applying the house opinions in [`../CLAUDE.md`](../CLAUDE.md) §3.

## Steps (traverse top-to-bottom; do not skip)
1. Build the standalone DCF — unlevered FCF, defensible WACC, terminal value, sensitivity.
2. Pull trading comparables — EV/EBITDA (± EV/Revenue) for a defensible peer set; cite each with source + date (§3 #8).
3. Pull precedent transactions — multiples paid in comparable deals (they embed a control premium); cite source + date.
4. Reconcile the three — where they diverge, explain the divergence; don't average it away (§3 #4).
5. Net synergies and integration cost — add only a defensible share of buyer-created synergies; subtract integration cost and control premium (§3 #2).

## Output
A triangulated valuation range with the divergence explained, the synergy logic, and a walk-away. See [`../skills/triangulate-a-valuation/SKILL.md`](../skills/triangulate-a-valuation/SKILL.md) and [`../knowledge/ma-valuation-and-deal-economics.md`](../knowledge/ma-valuation-and-deal-economics.md).

## Guardrails
- Never quote a current multiple from memory; cite source + date or mark `[unverified — training knowledge]` (§3 #8).
- Not a fairness opinion or a formal valuation; audited numbers route to accountants.
- End with owner / date / expected value impact.
