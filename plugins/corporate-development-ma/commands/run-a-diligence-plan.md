# Run a diligence plan

You are running `/corporate-development-ma:run-a-diligence-plan` for `$ARGUMENTS`. Run it the way the team's specialists would — applying the house opinions in [`../CLAUDE.md`](../CLAUDE.md) §3.

## Steps (traverse top-to-bottom; do not skip)
1. List the thesis assumptions — the value drivers and deal-breakers the deal depends on.
2. Map a workstream to each assumption — financial/commercial/tech/legal/people, with the confirm-or-kill question (§3 #5).
3. Read the quality of earnings — scrutinize add-backs, separate recurring from one-time; audited confirmation routes to accountants.
4. Underwrite people and culture — key-person concentration, retention, cultural fit (§3 #7).
5. Rank the red flags by impact — thesis impact + price impact, evidence needed, escalation owner.

## Output
A diligence plan (workstreams mapped to assumptions), a QoE read, a people/culture underwrite, and a ranked red-flag register with owners. See [`../skills/run-a-diligence-plan/SKILL.md`](../skills/run-a-diligence-plan/SKILL.md).

## Guardrails
- Every workstream tests a thesis assumption; drop the ones that don't (§3 #5).
- No MNPI you aren't authorized to hold; no legal/accounting conclusions (route out).
- End with owner / date / expected value impact on each red flag.
