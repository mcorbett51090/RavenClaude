# Frame a deal thesis

You are running `/corporate-development-ma:frame-a-deal-thesis` for `$ARGUMENTS`. Run it the way the team's specialists would — applying the house opinions in [`../CLAUDE.md`](../CLAUDE.md) §3.

## Steps (traverse top-to-bottom; do not skip)
1. Write the one-sentence thesis — why this / why now / how value. If you can't, stop (§3 #1).
2. Test buy vs build vs partner — would either achieve the outcome cheaper/faster?
3. Name the value-creation drivers — magnitude + cost-synergy vs revenue-synergy confidence.
4. Name the deal-breakers up front — the findings that kill the deal at any price.
5. State the walk-away — the price and conditions beyond which you walk.

## Output
A one-sentence thesis, a buy-vs-build/partner check, value-creation drivers, deal-breakers, and a walk-away. See [`../skills/frame-a-deal-thesis/SKILL.md`](../skills/frame-a-deal-thesis/SKILL.md) and [`../templates/deal-thesis-memo.md`](../templates/deal-thesis-memo.md).

## Guardrails
- No model before the thesis (§3 #1); no MNPI you aren't authorized to hold.
- Cite a source + date for every external figure (or mark it).
- End with owner / date / expected value impact.
