# Plan post-merger integration

You are running `/corporate-development-ma:plan-post-merger-integration` for `$ARGUMENTS`. Run it the way the team's specialists would — applying the house opinions in [`../CLAUDE.md`](../CLAUDE.md) §3.

## Steps (traverse top-to-bottom; do not skip)
1. Build the synergy register — every synergy has an owner, a date, and a cost-to-achieve; cost vs revenue separated (§3 #3).
2. Estimate dis-synergies and integration cost — one-time cost, attrition, disruption, timeline; feed back to valuation (§3 #6).
3. Design the operating model — integration depth per function + Day-1 readiness list.
4. Underwrite retention — key-person plan from diligence; package terms route to counsel (§3 #7).
5. Sequence the 100 days — operating model, systems, people, customers, with owners, dates, and synergy milestones.

## Output
A synergy register, an integration-cost estimate fed to valuation, an operating-model decision, a retention plan, and a 100-day plan. See [`../skills/plan-post-merger-integration/SKILL.md`](../skills/plan-post-merger-integration/SKILL.md) and [`../templates/integration-plan.md`](../templates/integration-plan.md).

## Guardrails
- No un-owned synergies; no deal underwritten on flawless, free integration (§3 #6).
- Retention/comp package terms route to counsel; employee PII stays out.
- End with owner / date / expected value impact.
