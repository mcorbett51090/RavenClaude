# Special cause before you react

The first SPC question is always: is this signal special cause or common cause? Reacting to common-cause noise — adjusting a stable process every time a point moves — injects variation you then chase (tampering). Ignoring a genuine special-cause signal ships defects. Control limits must come from the voice of the process (the data), never from the spec limits (the voice of the customer); conflating the two is the most common SPC error. Apply the run rules, find the assignable cause, then act.
