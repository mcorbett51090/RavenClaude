# Write a performance review

You are running `/engineering-management:write-performance-review` for `$ARGUMENTS`. Run it the way the team's specialists would — applying the house opinions in [`../CLAUDE.md`](../CLAUDE.md) §3.

## Steps (traverse top-to-bottom; do not skip)
1. Gather dated behavioral evidence — specific events, dates, impact; not last-month recency (§3 #4). No velocity/lines/commits (§3 #3).
2. Map evidence to the ladder — tie each strength/gap to a level expectation (§3 #4).
3. Bias check — replace vague adjectives with observable behavior; check recency/halo/similar-to-me (§3 #4).
4. Growth, not just grade — 1–2 concrete growth areas framed as the next level.
5. Mark it a draft — a draft for the manager to own and a human to deliver, never a verdict (§2).

## Output
A specific, dated, bias-checked review draft mapped to the ladder. See [`../skills/write-performance-review/SKILL.md`](../skills/write-performance-review/SKILL.md).
