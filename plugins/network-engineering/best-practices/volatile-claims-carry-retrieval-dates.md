# Volatile claims carry retrieval dates

**Rule:** Vendor/platform feature support, product names, SD-WAN/SASE capabilities, and standard-adoption claims carry a retrieval date and a re-verify-at-use rider. Durable engineering principles can be stated with confidence.

**Why:** the networking vendor landscape moves (acquisitions, renames, feature parity shifts). A confident-but-stale product claim in a client design is a credibility and correctness risk. The 2026 capability map is a starting point, not a citation.

**Anti-pattern:** quoting a vendor's current feature set or a "best SD-WAN" from memory with no date and no check against current docs before it lands in a design.
