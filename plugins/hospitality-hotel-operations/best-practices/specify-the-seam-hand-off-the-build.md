# Specify the seam, hand off the build

This plugin runs the rooms and the property; it does not run the F&B outlet, own the demand-forecast statistical method, or build the BI warehouse. When work crosses one of those seams, do the rooms-side slice fully, then specify the seam precisely — what is needed, in what shape, with what acceptance check — and hand the build to the right neighbour: the restaurant / bar / banquet / kitchen to `restaurant-operations`, the seasonality model and confidence intervals to `applied-statistics`, the reporting pipeline and dashboards to `data-platform`. A clean seam is an explicit interface, not a vague "and then F&B does their thing." Banquet/event revenue that touches both is split at the seam — the rooms block here, the catering there — never silently absorbed into one side.
