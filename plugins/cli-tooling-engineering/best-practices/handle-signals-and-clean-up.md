# Handle signals and clean up

A CLI gets interrupted — Ctrl-C (SIGINT), a `kill` (SIGTERM), a closed pipe (SIGPIPE), a parent shell exiting. A tool that ignores these leaves temp files, half-written output, a corrupted terminal (if it took the alt-screen or raw mode), or a held lock. Install handlers for **SIGINT/SIGTERM** that **clean up** — remove temp files, flush/finish or discard partial output atomically, release locks, and **restore the terminal** (show cursor, exit raw/alt-screen mode) — then exit with the conventional **`128 + signal`** code (SIGINT → 130) so the calling shell sees it was signalled. For SIGPIPE (the reader of your pipe went away, e.g. `tool | head`), exit quietly rather than crashing with a broken-pipe traceback. The principle: every exit path, including the involuntary ones, leaves the system as clean as a normal exit would.
