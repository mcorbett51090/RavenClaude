# Ship a --version flag and an update path

Every distributed CLI needs a `--version` that reports **exactly what's running** — and the only way to guarantee that is to **stamp the binary at build time** with the semver **and the commit SHA** (and ideally the build date). A `--version` that prints a hard-coded string the developer forgot to bump, or that omits the commit, turns every bug report into a guessing game about which build the user actually has. Pair it with a deliberate **update story**: for package-manager installs, document the upgrade command; for standalone binaries, consider an unobtrusive update *notice* (check at most occasionally, never block, never on a non-interactive run) or a verified self-update. The version and its update path are part of the tool's contract — they're how users and your own triage know what code is in the field.
