# Accreditation Evidence & Assessment

**Purpose:** make accreditation a continuous evidence system and a real assessment loop — so a review
is a report on work already done, not a last-minute fire drill.

> **Compliance note:** accreditation standards vary by accreditor and change on review cycles. Map to
> *your* accreditor's current standards; treat generic lists as `[verify against current standards]`.

---

## Steps

### 1. Map standards to evidence artifacts

For each accreditation standard, name the specific artifact(s) that demonstrate compliance (data,
policy, assessment results, minutes, syllabi). The map is the backbone — a standard with no mapped
artifact is a gap.

### 2. Run an assessment loop that closes

Assessment is only real if the loop closes:

```
define outcomes → measure → analyze → ACT (change something) → re-measure
```

The most common finding against institutions is assessment that measures but never acts. The
"close-the-loop" step is the one to protect.

### 3. Maintain evidence continuously

Keep the standards-to-artifact map current year-round, with owners and refresh cadence. Continuous
maintenance turns the self-study into assembly rather than archaeology.

### 4. Identify gaps early

Audit the map for standards with thin, stale, or missing evidence well before the review window.
Prioritize the highest-risk gaps (assessment loops that don't close, missing outcomes data).

### 5. Coordinate the FERPA-aware data flow

Student-level evidence carries FERPA constraints. Coordinate the data flow with the
academic-operations-and-compliance coordinator so evidence is gathered without violating
legitimate-educational-interest boundaries.

---

## Output

A standards-to-evidence map, an assessment-loop design that closes, and a prioritized gap list. Use
the [`accreditation-evidence-map`](../../templates/accreditation-evidence-map.md) template; deepen
with the [`compliance-and-accreditation-reference`](../../knowledge/compliance-and-accreditation-reference.md).
