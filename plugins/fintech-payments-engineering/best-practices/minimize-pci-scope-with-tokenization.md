# Minimize PCI scope with tokenization

Use the PSP's client-side tokenization so the raw card number travels directly from the browser to the processor and never touches your servers, leaving you holding only a token (SAQ-A posture). Every system that touches card data falls into PCI scope and must be assessed, so scope minimization — not adding controls to in-scope systems — is the dominant compliance strategy. The cheapest and safest card data to protect is the card data you never receive; never log a PAN or CVV.
