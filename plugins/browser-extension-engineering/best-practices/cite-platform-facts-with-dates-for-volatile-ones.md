# Cite platform facts; date the volatile ones

**Rule.** MV3 *mechanics* are durable and can be stated plainly. Store policies and
per-browser API availability are volatile — cite the source + date, or mark
`[unverified — training knowledge]` and verify (WebFetch/WebSearch) before the
claim enters a customer-facing deliverable or gates a submission.

**Why.** Store review requirements and API parity shift; asserting a stale policy
as current ships confidently-wrong platform facts — the failure mode the
marketplace accuracy discipline exists to prevent.

**Smell.** "The Chrome Web Store requires X" with no date or source.

**Cite:** AGENTS.md accuracy discipline; plugin §4.8.
