# Keep a single, honestly-described purpose

**Rule.** The extension does one clear thing; its permissions map to that thing;
its store listing matches its actual behavior. Make purpose, permissions, and
description agree before submitting.

**Why.** The fastest store rejections come from a mismatch between what the
extension does, asks for, and claims (single-purpose policy violations).

**Smell.** An extension bundling several unrelated features; a description that
over-claims relative to the permissions.

**Cite:** plugin §4.4; store rejection reasons in
`knowledge/cross-browser-and-stores.md`.
