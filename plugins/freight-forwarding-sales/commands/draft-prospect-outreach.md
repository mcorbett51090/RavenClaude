# Draft prospect outreach

You are running `/freight-forwarding-sales:draft-prospect-outreach`. Build outreach for the target the user described (`$ARGUMENTS`), using this plugin's `prospecting-outreach-strategist` discipline and the `prospect-outreach` skill.

## Steps
1. **Sharpen the ICP + trigger.** Name the shipper profile, the lane, the likely pain, and the **trigger event** (new facility, sourcing shift, hiring, lane disruption, peak). If no trigger is given, identify the most likely one to anchor on (and flag it as an assumption).
2. **Choose the shape** — a full ~8-touch multi-channel sequence (email/call/LinkedIn, optionally video) or a single personalized message + 2 follow-ups, per the ask.
3. **Write each touch on the Problem → Proof → Ask framework** — short, personalized opener; a specific lane proof (labeled illustrative if not the seller's own); a low-friction ask (benchmark / 15-min / single-lane trial).
4. **If they likely have a forwarder**, use the wedge (second source / benchmark / trial); never attack the incumbent by name.
5. **Include objection-handling** for the likely brush-offs (happy-with-current, no-time, send-an-email).
6. Emit the sequence/message in the Output Contract format + the Structured Output JSON block.

## Guardrails
- Personalize or don't send — every touch needs a trigger or lane-specific hook.
- Sell the outcome (reliability/visibility), not the rate; discount is the last lever.
- Land small (a wedge), don't ask a happy shipper to rip-and-replace.
- Label any proof point as the seller's real data vs illustrative.
