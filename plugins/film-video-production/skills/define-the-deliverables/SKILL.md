# Skill: Define the deliverables

A project that misses deliverables isn't delivered.

## Step 1 — List the deliverables
Masters, formats, versions, captions, and metadata (§3 #6).

## Step 2 — Define QC
The quality-control and acceptance criteria.

## Step 3 — Trace to the pipeline
Map each deliverable to a finishing step (§3 #3).

## Step 4 — Price the spec
Reconcile the deliverables to the budget (§3 #1).

## Output
A delivery-spec definition with QC, traced to the pipeline and the budget.
