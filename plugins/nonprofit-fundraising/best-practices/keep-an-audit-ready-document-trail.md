# Keep an Audit-Ready Document Trail

**Status:** Absolute rule
**Domain:** Nonprofit grants — post-award compliance
**Applies to:** `nonprofit-fundraising`

---

## Why this exists

An audit is not a special event you prepare for once; it is a state the grant file should be in continuously. The test is simple: can any charge on a grant be traced — in minutes — from the **general ledger → the source document → the approved budget line → the award terms**? If yes, the organization is audit-ready and a Single Audit or funder review is routine. If no, every audit becomes a frantic reconstruction, findings accumulate, and a missing time-and-effort record or an unsupported drawdown can turn an allowable cost into a disallowed one purely for lack of documentation. The cost of keeping the trail current is a filing habit; the cost of not keeping it is repayment, findings, and a high-risk designation that raises scrutiny on every future award. This is the documentation discipline that makes the allowable-cost and budget-vs-actual rules *defensible*. `[verify-at-use 2026-06-22 — records-retention period and audit requirements are set by 2 CFR 200 and the award]`

## How to apply

Maintain, per award, a retrieval-ready file containing:

```
☐ Executed award agreement + all amendments / approvals
☐ Approved budget + every revision
☐ Source document for every charge (invoice, receipt, timesheet, time-and-effort, contract)
☐ Drawdown requests + GL reconciliations + backup
☐ Every submitted report + its submission confirmation
☐ Cost-allocation methodology + indirect-rate basis (NICRA / de-minimis)
☐ Subrecipient determinations, subaward terms, and monitoring records (if any)
```

Keep the file for the **records-retention period** the award specifies (a multi-year window after the final report, set by 2 CFR 200 for federal awards — and extendable by litigation, audit, or claim). `[verify-at-use 2026-06-22 — confirm the period for each award]`

**Do:**
- File as you go — attach the source document at the moment of the charge, not at audit time.
- Make the trail traceable both directions (ledger ⇄ source ⇄ budget line ⇄ award).
- Use the audit-readiness checklist in [`../templates/grant-budget-vs-actual-tracker.md`](../templates/grant-budget-vs-actual-tracker.md).

**Don't:**
- Discard records at the project's end — the retention clock runs *after* the final report.
- Rely on memory for cost-allocation or split-salary bases; document the methodology.
- Let a subrecipient's documentation gap become yours — monitor and keep their records too.

## Edge cases / when the rule does NOT apply

Private-foundation grants may carry shorter or looser documentation terms than federal awards — but keep the agreement, budget, charges, and reports regardless; good documentation is never the wrong call. When an award's retention terms exceed the org's default policy, **the award terms govern**.
