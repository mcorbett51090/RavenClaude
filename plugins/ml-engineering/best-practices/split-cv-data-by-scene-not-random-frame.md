# Split CV data by scene, not random frame

Computer-vision data is rarely i.i.d. at the frame level: video frames, multi-shot product photos, tiled large images, and repeated captures of the same patient or object are near-duplicates of one underlying source. A random per-frame train/test split scatters near-identical frames across both sides, leaking the answer into the test set and inflating every metric — the model looks excellent offline and disappoints in production on genuinely unseen scenes. Split by the **source unit (scene / video / capture session / patient / object)** so an entire source lands wholly in one split, and check for near-duplicates across the boundary. This is the CV-specific face of validate-without-leakage: an offline metric earned on a leaked split is a promise production cannot keep.
