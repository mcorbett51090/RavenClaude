# The registry is the source of truth for what ships

A deployed model is a specific, registered version with known lineage — its training data, code, metrics, and approval state — not a file someone copied to a server or a pickle handed over in Slack. Promotion to staging or production happens by moving a version through the registry's stages, and serving pulls from the registry, so that "what is in production right now and how was it built" always has a single, queryable answer. Without that, the production model is whatever artifact happened to land on the box, rollback is a scramble to find the previous file, and an audit has no trail. The registry is the gate every model passes through; if a model can reach production without going through it, you don't have a source of truth, you have a convention people break under deadline.
