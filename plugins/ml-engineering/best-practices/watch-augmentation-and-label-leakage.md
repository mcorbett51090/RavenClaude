# Watch augmentation and label leakage

Augmentation is essential in computer vision but is a leakage and correctness trap. Augment the **training split only, after the split is made** — augmenting before splitting puts transformed copies of the same source image on both sides, textbook leakage that inflates the offline metric. Never augment the validation/test set; it must mirror real production inputs (test-time augmentation is a separate, deliberate inference technique, not leaked eval copies). Use only **label-preserving** transforms: a horizontal flip can be label-destroying for OCR/text, asymmetric poses, or left/right-handed classes, and any geometry change (rotation, crop, scale) must carry the box/mask/keypoint with it or the annotation is now wrong. And don't augment past the deployment distribution — invariances the inference target will never see can hurt. Honest augmentation widens coverage; careless augmentation manufactures a metric the world won't honor.
