# Track every experiment, or the result is undefendable

Log the parameters, metrics, code version, data version, and environment for every training run, automatically, so any result can be reproduced and any two runs can be compared on equal footing. An untracked experiment is a number you cannot defend: when someone asks why the new model is better, "I think it was the bigger learning rate, but I also changed the features that day" is not an answer, it's an admission that you were tuning in the dark. Experiment tracking is cheap to add up front and impossible to reconstruct after the fact — the run that produced your best model is gone the moment the notebook kernel restarts. Make tracking the default path of the pipeline, not a discipline you remember to apply, so the comparison the statistician needs is always available.
