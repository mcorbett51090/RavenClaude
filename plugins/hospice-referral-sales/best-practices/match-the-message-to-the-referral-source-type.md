# Match the message to the referral-source type — or don't send it

**Status:** Absolute rule
**Domain:** Outreach / messaging
**Applies to:** `hospice-referral-sales`

---

## Why this exists

A discharge planner, a SNF director of nursing, a hospitalist, an oncologist, and an ACO medical director have different drivers — readmission penalties, survey measures, throughput, continuity, total cost of care. A generic "we provide compassionate end-of-life care" message speaks to none of them and reads as noise. The message has to lead with the *specific source's* pressure and the patient-access value relevant to their population, anchored to a trigger event. Personalize to the source type, or don't send.

## How to apply

**Do:**
- Open with the source type's own pressure and patient population.
- Anchor to a trigger event (a new palliative program, a readmission penalty, a staff change).
- Lead with patient access and the source's outcome, not the agency.
- Use a different angle for each segment.

**Don't:**
- Send the same generic message to every source type.
- Lead with "please refer to us."
- Skip the trigger event and the source-specific value.

## Edge cases / when the rule does NOT apply

A genuinely universal, low-frequency announcement (a new after-hours admission capability that helps every source) can go broadly — but even then, the framing leads with the source's benefit, not the agency's.
