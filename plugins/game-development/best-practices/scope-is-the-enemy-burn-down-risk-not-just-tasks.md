# Scope is the enemy — burn down risk, not just tasks

**Status:** Pattern. **Constitution:** §3, §4.

## Use when
Any game development deliverable where this question is in play.

## The rule
Most game projects fail on scope, not talent; production tracks the riskiest unknowns (fun, tech, content cost) to burn down first, not just a task list.

## Why
This is house opinion §3 #3. Getting it wrong is the most common way a game development analysis misleads the operator who acts on it.

## How to apply
- Apply it before reaching for a method.
- Make the relevant number carry a definition, window, and baseline (§3 #1).
- Cite the source + date for any external figure, or mark it `[unverified — training knowledge]`.
