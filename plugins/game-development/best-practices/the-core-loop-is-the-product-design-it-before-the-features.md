# The core loop is the product — design it before the features

**Status:** Absolute rule. **Constitution:** §3, §4.

## Use when
Any game development deliverable where this question is in play.

## The rule
Retention lives in the second-to-second and session-to-session loop; features layered on a weak loop don't save it. Design the loop, then the meta.

## Why
This is house opinion §3 #2. Getting it wrong is the most common way a game development analysis misleads the operator who acts on it.

## How to apply
- Apply it before reaching for a method.
- Make the relevant number carry a definition, window, and baseline (§3 #1).
- Cite the source + date for any external figure, or mark it `[unverified — training knowledge]`.
