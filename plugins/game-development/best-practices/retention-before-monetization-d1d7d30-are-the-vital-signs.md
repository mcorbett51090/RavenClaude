# Retention before monetization — D1/D7/D30 are the vital signs

**Status:** Pattern. **Constitution:** §3, §4.

## Use when
Any game development deliverable where this question is in play.

## The rule
A game that doesn't retain can't monetize; early retention (D1/D7/D30) is the first gate, and monetization design follows a retaining loop, not the other way around.

## Why
This is house opinion §3 #4. Getting it wrong is the most common way a game development analysis misleads the operator who acts on it.

## How to apply
- Apply it before reaching for a method.
- Make the relevant number carry a definition, window, and baseline (§3 #1).
- Cite the source + date for any external figure, or mark it `[unverified — training knowledge]`.
