# A control has three states — designed, implemented, operating-effectively

"We have a policy" describes a control that is only *designed*. A control is *implemented* when it is actually running, and *operating-effectively* only when there is evidence it ran across an observation period without exception. Type II reports and ISO certification require the third state, not the first. Always name which state a control is in, what evidence supports the claim, and what closes the gap — conflating a written policy with a working control is the most common source of audit findings.
