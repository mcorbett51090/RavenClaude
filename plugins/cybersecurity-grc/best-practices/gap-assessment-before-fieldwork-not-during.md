# Run the gap assessment before fieldwork, not during

The cheapest finding is the one you find yourself. A gap assessment run *before* the observation period starts — against the actual framework criteria, with the same sampling rigor the auditor will use — turns would-be audit exceptions into remediation items you control the timing of. Discovering a missing access-review cadence in a self-assessment three months early means you stand up the control and accrue a clean evidence window; discovering it in fieldwork means a disclosed exception in the report. Assess design first (is the control specified and owned), then operating effectiveness (has it run with evidence over a representative window), and treat any gap as a remediation with an owner and a date — not a note. The pre-audit gap assessment is also where you confirm the scope is attestable and the evidence is collectible at the source before the clock starts.
