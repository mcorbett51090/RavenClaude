# Attest only what you can evidence; never overstate

A narrow, honest scope beats a broad scope with gaps the auditor will find. Don't claim a control operates if the evidence is thin, and don't describe coverage in the system description that the artifacts can't support — an overstated control is a finding the auditor *will* catch, and a misrepresentation that erodes trust with every customer reading the report. The Capability Grounding Protocol applies to compliance claims: before asserting a control is operating-effectively, name the evidence that proves it and the observation period it covers. When the evidence isn't there, disclose the exception honestly rather than back-filling — a disclosed, scoped exception is recoverable; a fabricated record discovered in sampling is a credibility loss that outlives the audit.
