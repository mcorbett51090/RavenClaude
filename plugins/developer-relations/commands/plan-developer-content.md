# /plan-developer-content

Spawn `developer-advocate` (with `developer-marketing-and-growth-strategist` for positioning) to
build a developer content plan that converts.

## What it does

1. Sets the positioning (job-to-be-done, alternative, differentiated path) before topics.
2. Tags each piece to a journey stage (awareness / evaluation / activation / adoption) and a goal metric.
3. Chooses the format that fits the stage, not the trend.
4. Designs the activation path that ends each artifact.

## Usage

```
/plan-developer-content
```

Then share your audience, product, and current content (if any). The agent applies
[`developer-content-and-advocacy`](../skills/developer-content-and-advocacy/SKILL.md) and the
[`content-plan-by-journey-stage`](../templates/content-plan-by-journey-stage.md) template.

## Good inputs

- Who you're reaching (IC builder / tech lead / platform owner) and the job-to-be-done.
- Where the funnel currently leaks (so content targets the right stage).
