# Plan DevRel content

You are running `/developer-relations:plan-devrel-content`. Build the content plan
for `$ARGUMENTS` using the `developer-advocate` discipline: format follows the
activation goal, and DevRel measures activation, not MQLs.

## Steps

1. **Name the activation goal first** — which funnel step (first success / value
   path / retention) this content moves. No goal → no content.
2. **Pick the format from the goal** (tree 3 in
   [`../knowledge/devrel-engagement-decision-trees.md`](../knowledge/devrel-engagement-decision-trees.md)):
   getting-started for first success, sample app for the value path, guide for a
   concept, talk/post/video for reach.
3. **Fix the getting-started path before reach** — reach onto a broken first-run is
   wasted.
4. **Write engineer-to-engineer** — no marketing-speak; show the code.
5. **Build the calendar** in the
   [`devrel-content-calendar`](../templates/devrel-content-calendar.md) template,
   each item naming its activation goal and target metric.

## Guardrails

- Reject content with no activation goal ("we should blog more").
- Route demand-gen/brand/lead-gen asks to `marketing-operations`.
