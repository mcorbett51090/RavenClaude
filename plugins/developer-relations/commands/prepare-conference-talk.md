# /prepare-conference-talk

Spawn `developer-advocate` to shape a CFP submission and a talk that earns activations, not just
applause.

## What it does

1. Picks the funnel stage the talk serves (usually awareness/evaluation) and a problem-first angle reviewers accept.
2. Outlines the talk around the audience's job-to-be-done (problem → journey → technique → runnable proof).
3. Designs the post-talk activation path (repo, quickstart, sandbox).
4. Sets the measurement plan (post-talk repo visits, quickstart starts, attributable activations).

## Usage

```
/prepare-conference-talk
```

Then share the conference, audience, and the technique/story you want to tell. The agent applies
[`conference-talk-and-cfp-strategy`](../skills/conference-talk-and-cfp-strategy/SKILL.md) and the
[`conference-talk-brief`](../templates/conference-talk-brief.md) template.

## Good inputs

- The specific developer problem and the concrete takeaway.
- A real number or hard-won lesson the talk can anchor on.
