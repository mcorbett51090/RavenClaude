# Draft a CFP abstract

You are running `/developer-relations:draft-cfp-abstract`. Write an accept-able, attendee-led CFP
abstract for `$ARGUMENTS` — the discipline the `conference-talk-and-cfp` skill enforces.

## When to use this

Submitting a talk to a developer conference, or rescuing a product-pitch abstract. NOT for a vendor
keynote at your own event (different norms).

## Steps

1. **Reframe to the attendee takeaway** — what will they be able to *do* after 30 minutes? The
   product, if present, is the example, not the subject (`cfp-abstract-leads-with-the-attendee-takeaway.md`).
2. **State 2–4 concrete takeaways**, each an action ("build/diagnose/avoid X"), never "understand X."
3. **Match the track and level**; write reviewer-fit notes.
4. **Right-size the scope** for the slot — one clear arc.
5. **Fill the template** ([`../templates/cfp-abstract.md`](../templates/cfp-abstract.md)) and run the
   first-sentence test (the abstract must not open with "At <Company>…").

## Guardrails

- If the first sentence is about you or the product, rewrite it.
- "Learn about" is not a takeaway — name the action.
- Don't over-scope; reviewers reject talks that promise more than the slot allows.
