# developer-relations — best-practices library

Codified house opinions for the `developer-relations` plugin. Each file is one rule the agents
enforce, with why-it-exists, how-to-apply, edge cases, and provenance. They map to the §3 house
opinions and §4 anti-patterns in [`../CLAUDE.md`](../CLAUDE.md).

| File | Rule |
|---|---|
| [`measure-developer-activation-not-vanity-metrics.md`](measure-developer-activation-not-vanity-metrics.md) | Report activation/time-to-first-success, not stars/followers/registrations. |
| [`optimize-time-to-first-success.md`](optimize-time-to-first-success.md) | TTFS is the highest-leverage surface; count the steps and cut them. |
| [`sample-code-must-run-as-shipped.md`](sample-code-must-run-as-shipped.md) | Every sample runs unmodified from clean; no placeholder secrets/TODOs on the happy path. |
| [`teach-dont-market-at-developers.md`](teach-dont-market-at-developers.md) | The unit of DevRel helps a developer do their job today; strip the marketing-speak. |
| [`cfp-abstract-leads-with-the-attendee-takeaway.md`](cfp-abstract-leads-with-the-attendee-takeaway.md) | Lead a talk proposal with what the attendee can do afterward, not the speaker/product. |
| [`community-health-is-response-time-and-resolution.md`](community-health-is-response-time-and-resolution.md) | Measure a community by response time + resolution, not member count. |
| [`volatile-claims-carry-retrieval-dates.md`](volatile-claims-carry-retrieval-dates.md) | Reach numbers, tooling/pricing, and CFP deadlines carry a retrieval date and are re-verified. |
