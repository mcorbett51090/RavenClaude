# Developer Content & Advocacy

**Purpose:** produce developer content and advocacy that moves a developer toward value, not content
that merely informs — every piece tagged to a journey stage, a goal, and a next step.

---

## Steps

### 1. Tag every piece to a journey stage and a goal

Before writing anything, place the piece on the journey and give it a measurable goal:

| Stage | Developer's question | Format that fits | Goal metric |
|---|---|---|---|
| Awareness | "Does this solve a problem I have?" | Conceptual post, conference talk, comparison | Reach → quickstart starts |
| Evaluation | "Will this work for my case?" | Tutorial, sample app, reference architecture | Sign-ups, trial starts |
| Activation | "How do I get this working?" | Quickstart, runnable demo, workshop | Time-to-first-value, activation rate |
| Adoption | "How do I run this in production?" | Deep guide, best-practice doc, case study | Production adoption, expansion |

A topic with no stage and no goal is a wishlist item, not a plan.

### 2. Choose format by stage, not by trend

Match the format to the developer's question at that stage. A lightning talk does not activate; a
quickstart does not create awareness. Resist doing the trendy format for the wrong stage.

### 3. Design the activation path

Every artifact ends with the next concrete step toward value — a repo, a quickstart link, a
sandbox, a CLI command. A talk or post that informs but leaves no path converts nothing.

### 4. Make demos survive a real developer

No magic setup, no pre-warmed state, no happy-path-only. Declare versions, use real commands, and
show at least one realistic failure-and-recovery. A demo a developer can type along with builds
trust; a demo that "just works" by sleight of hand destroys it.

### 5. Build the feedback loop

Define what you'll listen for at events and in threads, the capture format, and the routing path
into product. Structured developer signal is DevRel's unique asset.

---

## Output

A content plan (each item: stage, format, goal, activation path), a CFP angle + talk outline, or a
demo spec with its failure modes. Compute content ROI and funnel conversion with
[`../../scripts/devrel_calc.py`](../../scripts/devrel_calc.py).
