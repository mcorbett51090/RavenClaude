# Make forms controlled and validate at the boundary

A form is the app's main write path, so treat it as one. Prefer a form library (React Hook Form, TanStack Form) over hand-rolled `useState` per field — you get validation, dirty/touched tracking, and submission state without re-rendering the whole form on every keystroke. Validate with a schema (Zod) that is the single source of truth shared between client and the typed API boundary, so the client check and the server check can't disagree. Surface errors per field with accessible associations (`aria-describedby`, `aria-invalid`), disable submit only on a real reason, and never trust client validation as security — it's UX; the server validates again. Uncontrolled inputs scattered with ad-hoc `onChange` handlers are where stale, unvalidated, inaccessible forms come from.
