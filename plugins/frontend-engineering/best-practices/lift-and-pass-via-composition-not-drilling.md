# Solve prop-drilling with composition, not a global store

When a prop threads through five layers that don't use it, the fix is usually composition, not context and rarely a global store. Pass the rendered element as `children` or a slot so the data-owning parent composes the consumer directly and the intermediate layers never see the prop. Reach for context only for genuinely tree-wide, low-churn values (theme, current user, locale) — context re-renders every consumer on change, so high-frequency state there causes diffuse jank. A global state store is for truly cross-cutting client state, not for dodging a prop that three components in one subtree share. Choosing a store to avoid drilling couples unrelated parts of the app and makes the data flow impossible to follow.
