# Request the narrowest extension permissions that work

Start a browser extension at the least-privileged permission tier and escalate only when a concrete feature forces it: prefer `activeTab` (temporary current-tab access on a user gesture, no install warning) over host permissions; prefer a few specific `host_permissions` match patterns over `<all_urls>`; and declare broad access as `optional_permissions` requested at runtime with `chrome.permissions.request()` rather than at install time. Every permission you request is a user-trust cost, a larger attack surface if the extension is compromised, and a slower, stricter store review — so re-audit at every release and drop any permission no current feature uses. Broad permissions and any remotely-hosted code are the top drivers of store-review rejection.
