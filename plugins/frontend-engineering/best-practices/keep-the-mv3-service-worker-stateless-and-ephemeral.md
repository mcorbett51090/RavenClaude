# Keep the MV3 service worker stateless and ephemeral

The Manifest V3 background is an event-driven service worker that the browser terminates when idle and restarts on the next event — it is not the persistent background page of MV2. Treat it as stateless: register every event listener at the top level synchronously on each load (never inside an async callback, or the wake-up event is missed), hold no in-memory state you cannot afford to lose, and persist anything durable to `chrome.storage`, re-hydrating on wake. It has no DOM and no `localStorage`. Designing long-running or stateful logic in the service worker is the most common MV3 bug; if a feature feels like it needs a persistent background, move the state to storage and make the worker re-derive it on each event.
