# Isolate content scripts from the page and validate every cross-world message

A content script runs in an isolated JavaScript world: it can read and modify the page's DOM but not the page's own JS variables or globals, and the page cannot see the content script's. Keep it there whenever possible — the isolation is what prevents collisions with the page and limits the blast radius if the page is hostile. Only when you genuinely need the page's real `window` should you inject a main-world page script, and that script has no extension privileges and no isolation, so treat it as untrusted. The single bridge between the isolated and main worlds is `window.postMessage`; validate `event.origin`, `event.source`, and the message shape before acting on anything that crossed it — and apply the same untrusted-input discipline to all `chrome.runtime` / `chrome.tabs` messages, since an MV3 service worker may have been asleep and cannot rely on prior in-memory state.
