# Screen every applicant by the same written criteria

**Status:** Absolute rule
**Domain:** Fair-housing compliance / screening policy
**Applies to:** `property-management-residential`

---

## Why this exists

Inconsistent screening — approving one applicant and denying another with similar qualifications,
or applying criteria post-hoc to a specific application — creates disparate treatment liability
under the Fair Housing Act. Even when there is no discriminatory intent, inconsistent application
of criteria can produce a disparate impact on a protected class and expose the property manager to
an enforcement action.

The defense is procedural: written criteria that are set before any application is received, applied
identically to every applicant for the same property, and documented in the tenant file for every
decision.

## How to apply

1. **Write the criteria before advertising the unit.** Income multiple, credit score floor, rental
   history standards (eviction record, prior landlord reference, prior late payment history), and
   criminal history policy must be written and approved before the listing goes live.
2. **Apply the same criteria to every applicant.** The income multiple that qualifies one applicant
   must be the same standard used to evaluate every other applicant for the same property.
3. **Document the evaluation.** For every application, record the outcome against each criterion.
   If denied, retain the denial rationale and issue an adverse action notice per FCRA if the denial
   was based in whole or in part on a consumer report.
4. **Review criteria periodically.** Criteria should be reviewed at least annually or after any
   enforcement action to ensure they remain appropriate and do not create unnecessary disparate
   impact.

**Do:**

- Set written criteria (income, credit, rental history, criminal history) before taking the first
  application.
- Use a standard application form for all applicants at the same property.
- Retain the evaluation memo for every application (approved and denied) in the tenant file.
- Apply any flexibility (payment arrangements, co-signers, higher deposit) consistently — if you
  offer it to one, document why, and apply the same standard to all.

**Don't:**

- Approve an applicant who doesn't meet the written criteria without documenting the exception and
  applying the same exception consistently.
- Deny an applicant based on criteria that weren't in the written policy at the time of application.
- Apply different income multiples or credit standards to different applicants without a documented,
  non-discriminatory basis.
- Use a blanket criminal history ban without an individualized assessment provision (see HUD 2016
  guidance on criminal history and disparate impact).

## Edge cases / when the rule does NOT apply

- **Unit-specific criteria differences:** it is permissible to set different criteria for different
  unit types (e.g., income multiple based on rent level) if the criteria are set in advance and
  applied consistently to all applicants for that unit type.
- **Legitimate, documented exceptions:** a co-signer, a higher security deposit, or a guarantor
  can bridge a gap in income or credit — but only if the exception is consistently offered to all
  qualifying applicants, not selectively to some.
