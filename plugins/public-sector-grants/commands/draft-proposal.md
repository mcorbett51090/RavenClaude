You are running `/public-sector-grants:draft-proposal`. Use `proposal-writer` + the `proposal-narrative-and-budget` skill.

## Steps
1. Map the narrative section-by-section to the NOFO's review/scoring criteria, in the funder's order and language; flag any criterion the concept doesn't yet address (route back to grant-strategist if it's a strategy gap).
2. Write the needs/problem statement on evidence — baseline, gap, cited source — not adjectives.
3. Turn the logic model into goals and SMART objectives (each with baseline, target, measure, deadline).
4. Design the evaluation plan from the objectives: process + outcome indicators, data sources, cadence, who analyzes.
5. Build the budget + budget narrative — every line justified, tied to a logic-model activity, table and narrative agreeing to the dollar; call out the indirect rate and match. Route allowability questions to grants-compliance-analyst before any cost goes in.
6. Emit the proposal draft + the Structured Output block (with `Funder requirement traced:` and `Handoff:`).
