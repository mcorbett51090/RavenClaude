# Proposal Narrative & Budget

## Map the narrative to the review criteria
The NOFO/RFP scoring rubric is the structure. Write section-by-section against each weighted criterion, in the funder's order and language. A brilliant narrative that doesn't map to a criterion scores zero on it — points come from addressing what the funder weights, then making it read well.

## Build the needs/problem statement on evidence
Establish the problem with a baseline number, the gap, and a cited source before proposing the solution. "A pressing need" persuades no one; an unsourced statistic is worse than none. The need is evidence, not adjectives.

## Write SMART goals and objectives
Each objective is Specific, Measurable, Achievable, Relevant, Time-bound — with a baseline, a target, a measure, and a deadline, traced to the logic model. "Improve outcomes" is a wish; "+15% by 2028, state assessment" is an objective a reviewer can score and an evaluator can prove.

## Design the evaluation plan from the objectives
Every objective implies the data you'll collect to demonstrate it. Pair process measures (did we do it) with outcome measures (did it work); name the data sources, the collection cadence, and who analyzes. The evaluation plan must measure exactly what the objectives promised.

## Build the budget + budget narrative
Every line is justified in the narrative, tied to a logic-model activity, and flagged for the allowable/allocable/reasonable test. The budget table and narrative agree to the dollar. Call out the indirect-cost rate and match/cost-share. Route allowability questions to `grants-compliance-analyst` before a cost goes in — don't write an unallowable line.

## Output
A narrative mapped to the review criteria, a data-backed needs statement, SMART goals/objectives, an evaluation plan, and a budget + budget narrative tied to the logic model. Hand allowability/indirect-rate/match to `grants-compliance-analyst`; the GL reconciliation to `finance`.
