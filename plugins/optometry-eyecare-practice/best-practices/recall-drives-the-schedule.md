# Recall drives the schedule

**Status:** Absolute rule
**Domain:** Operations / scheduling
**Applies to:** `optometry-eyecare-practice`

> Advisory operations rule. Recall *intervals* are clinical-protocol decisions — `[verify-at-use]`. No PII/PHI.

---

## Why this exists

A routine eye-care practice lives or dies on recare. Marketing buys new patients; **recall keeps the ones you already have**, and a recall-fed schedule is predictable in a way a walk-in-fed one never is. A soft schedule is almost always a recall problem, not a capacity problem — which is why you work recall before adding a lane or a tech.

## How to apply

- Set the recall interval **by exam type** per clinical protocol (`[verify-at-use]`) — routine, contact-lens, medical follow-up, pediatric all differ.
- Make the recall list the **primary** schedule-filler; walk-ins fill the gaps, not the plan.
- Track recall reactivation rate as the schedule's leading indicator.
- Before adding capacity, confirm the schedule is actually full (capacity = lanes × slots × fill rate).

**Do:** work the recall list first; read fill rate before adding lanes.
**Don't:** let the recall list age while chasing walk-ins; add capacity to fix a fill-rate gap.

## Edge cases / when the rule does NOT apply

A brand-new practice with no patient base to recall is genuinely acquisition-led until it builds one — recall becomes the engine once the base exists.
