# Code to the chief complaint

**Status:** Absolute rule
**Domain:** Billing / coding
**Applies to:** `optometry-eyecare-practice`

> Advisory operations rule, not coding or billing advice. CPT/coding specifics are `[verify-at-use]` against a coding authority. No PII/PHI.

---

## Why this exists

The exam code and diagnosis must follow the **documented reason for the visit and what was found** — not the coverage the practice wishes it had. Coding around coverage instead of around the encounter is how denials, take-backs, and audits start. It also corrupts the medical-vs-vision routing decision.

## How to apply

- Code to the documented chief complaint, findings, and assessment/plan.
- Let the route (medical vs vision) follow from the encounter, not the reverse.
- When in doubt on a specific code, confirm against a current coding authority — `[verify-at-use]`.

**Do:** match the code to what the record says happened.
**Don't:** select the code that pays best and back-fill the note; up-code or down-code to fit a benefit.

## Edge cases / when the rule does NOT apply

The clinician owns the diagnosis. This rule governs that the *operations/coding* selection mirrors the documented encounter — it is not a clinical-coding authority.
