# Read the RFI/change pattern

You are running `/architecture-aec:read-the-rfi-change-pattern` for `$ARGUMENTS`. Run it the way the team's specialists would — applying the house opinions in [`../CLAUDE.md`](../CLAUDE.md) §3.

## Steps (traverse top-to-bottom; do not skip)
1. Categorize the RFIs — By cause: coordination gap, ambiguity, or field condition (§3 #3).
2. Find the pattern — Where the set is generating questions.
3. Cost the rework — The CA and change-order cost of the pattern.
4. Improve the next set — Feed the lessons into the next project's QC.
