# Layer controls, but cut the ones that only look like security

Defense in depth means independent layers that each fail closed, so one bypass isn't a breach. Security theater is a control that costs effort and trust without raising attacker cost — a 90-day password rotation that produces `Password1!`→`Password2!`, a WAF rule list nobody tunes, a checkbox audit that never changed a deploy. The test for a control: name the specific threat it raises the cost of and how you'd know it fired. If you can't, it's theater — and theater is worse than nothing because it consumes the budget and attention real controls need, and breeds the "we're compliant so we're secure" complacency. Prefer fewer controls that each demonstrably move attacker cost over a thick stack of unverified ones.
