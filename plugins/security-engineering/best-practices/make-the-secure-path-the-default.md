# Secure by default; insecure must be a deliberate opt-out

If using a system securely requires reading the docs and flipping switches, most deployments will be insecure — people ship the defaults. Invert it: encryption on, public access off, TLS required, least-privilege role, audit logging enabled — all out of the box, with the *insecure* option being the one that takes a conspicuous, logged, justified opt-out. This applies to the platforms you build as much as the ones you buy: a shared library, an internal service template, or a Terraform module should hand the next engineer a safe configuration they have to actively weaken, not a footgun they have to actively defuse. The most scalable security control is the default value, because it protects every user who never thinks about security at all.
