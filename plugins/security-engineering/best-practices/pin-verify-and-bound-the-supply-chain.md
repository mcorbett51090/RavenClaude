# Pin it, verify it, and bound what it can do

A dependency is code you didn't write running with your privileges, so trust it deliberately. Pin to immutable identifiers — a lockfile with hashes, an image by digest, a third-party CI action by commit SHA, never a floating tag a maintainer (or attacker) can move under you. Verify provenance where the chain supports it (SLSA attestation, signature on the artifact) rather than trusting that a name maps to the bytes you expect. Then bound the blast radius: a build-time dependency shouldn't need network egress, a transitive package shouldn't be able to read your secrets. Pinning without verification stops accidental drift but not a compromised-at-source release; verification without bounding still gives a trusted-but-malicious dependency the keys. You need all three.
