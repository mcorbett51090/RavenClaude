# Assume breach — design so one compromise isn't total

Perimeter security assumes the inside is trusted; that assumption is how a single phished credential becomes a full data exfiltration. Design as if the attacker is already inside: segment the network so lateral movement is bounded, authenticate and authorize service-to-service calls (don't trust a packet because it came from the VPC), encrypt data in transit *and* at rest so a stolen disk or sniffed link yields ciphertext, and instrument for detection because you will need to spot the intruder you assumed got in. The practical yardstick is blast radius — for each credential, host, and service, ask "if this is owned, what else falls?" and shrink that answer. Assume-breach isn't pessimism; it's refusing to bet everything on the perimeter holding.
