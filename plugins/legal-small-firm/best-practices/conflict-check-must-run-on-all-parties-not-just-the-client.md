# Conflict Check Must Run on All Parties, Not Just the Client

**Status:** Absolute rule
**Domain:** Small-firm legal practice
**Applies to:** `legal-small-firm`

---

## Why this exists

A conflict check that names only the prospective client misses the most common conflict pattern in small-firm practice: a conflict with an opposing party, a related entity, a guarantor, or a third party who is adverse in the matter. An attorney who represents a business owner on a contract dispute without checking whether the entity, its principals, or the counterparty have any history with the firm is not running a real conflict check — they are running a name search that creates false confidence. The resulting disqualification motion, ethics complaint, or malpractice claim is preventable.

## How to apply

Every conflict check must search, at minimum, the following party set before the intake call:

```
Conflict Check Party Matrix
─────────────────────────────
Matter type:  ____________

Parties to check:
  [ ] Prospective client (full legal name + any DBA / prior names)
  [ ] Related entities (parent, subsidiary, affiliate, guarantor, co-signer)
  [ ] Adverse party (counterparty, opposing counsel's client in the matter)
  [ ] Opposing counsel (firm-level check for screens)
  [ ] Third parties with a material interest (estate beneficiaries, co-owners, etc.)

Check sources:
  [ ] Client/matter management system (all matters, open + closed)
  [ ] Email archive search on all party names
  [ ] Attorneys' personal recollection (circulate the list to all attorneys)

Result:
  [ ] Clear — no conflict found
  [ ] Potential conflict — requires analysis under MRPC 1.7 / 1.9 / 1.10
  [ ] Conflict — decline or obtain informed consent per applicable rule
```

**Do:**
- Run the check before the intake call, not during it — an attorney who receives material facts before a conflict is identified may be imputed with knowledge that triggers a conflict.
- Document the check result and the party list in the matter file, date-stamped.
- Supplement the check when new adverse parties or interests appear mid-matter.

**Don't:**
- Run the check only against the client's name and declare it clear.
- Rely on memory alone for adverse-party history — search the system.
- Accept a new matter while the conflict result is pending; hold work until the check completes.

## Edge cases / when the rule does NOT apply

A truly prospective inquiry (a potential client who has not yet provided material facts about the matter) may receive limited information before the check — but no facts sufficient to create imputed knowledge should be taken until the check clears or is analyzed.
