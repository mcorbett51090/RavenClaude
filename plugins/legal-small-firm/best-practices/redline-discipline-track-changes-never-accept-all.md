# Redline Discipline — Track Changes, Never Accept-All

**Status:** Absolute rule
**Domain:** Small-firm legal practice
**Applies to:** `legal-small-firm`

---

## Why this exists

Accepting all changes in a contract redline without a provision-by-provision review is how material terms slip into a document unnoticed. In a small firm under time pressure, the accept-all temptation is real — the counterparty has returned a clean document and the client wants to close. The risk is that a quietly-inserted liability cap change, a definition tweak, or an indemnification expansion changes the deal's economics or risk profile without the attorney registering it. The client's claim against the firm for missing a negotiated-away protection is the outcome.

## How to apply

Every contract redline review follows a three-pass protocol:

```
Redline Review Protocol
────────────────────────
Pass 1 — Track-changes view (required)
  Never accept all before completing this pass.
  Review every marked change:
  [ ] Identify who inserted each change (counterparty markup vs your prior change)
  [ ] Note: definition changes, liability/indemnity provisions, termination rights,
            IP ownership, payment terms, dispute resolution

Pass 2 — Clean-view read
  After reviewing all marks, accept-all only to perform a full clean read.
  Confirm the document reads as intended with all changes incorporated.
  Check: cross-references, defined terms that shifted meaning, any inadvertent gaps.

Pass 3 — Issue log
  Document any provision accepted over the client's preferred position:
  | Provision | Change accepted | Client advised? | Date |
  |-----------|-----------------|-----------------|------|
  | [§X]      | [description]   | [Y/N]           |      |

```

**Do:**
- Maintain a running issue log for multi-round negotiations — memory degrades across rounds.
- Advise the client in writing of any accepted compromise from their preferred position, and get acknowledgment.
- Check defined-term changes as carefully as operative-provision changes — a redefined "Material Adverse Effect" can change the entire agreement's meaning.

**Don't:**
- Accept all before reviewing all marks — even when the counterparty says "just cleanup."
- Skip the clean-view read after accepting — track-changes view can obscure the full effect of combined edits.
- Send a marked draft without first confirming that your own prior changes are still present and unaltered.

## Edge cases / when the rule does NOT apply

A form document where the counterparty's redlines are only typographical corrections (spacing, formatting, no substantive changes) may be reviewed in a single pass after visual confirmation — but the "substantive" judgment still requires opening track-changes, not eyeballing the clean version.
