# Test the riskiest assumption first, and test it cheap

Every idea rests on a stack of assumptions — that the problem is real, that customers want this solution, that they'll pay, that you can build it — and the one most likely to be false and most fatal if it is should be tested before you write production code. Enumerate the assumptions, rank them by how much they'd hurt if wrong times how unsure you are, and attack the top one with the cheapest experiment that could falsify it: a fake door, a concierge run, a prototype, five interviews. Building first and validating later inverts the cost curve — you discover the killer assumption was false after spending the quarter. The riskiest-assumption test exists to fail fast on the cheap thing so you never fail slow on the expensive one.
