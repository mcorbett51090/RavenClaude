# Understand why a value is missing before you impute it

Imputation is a modeling decision, not a cleanup step — ask *why* a value is missing before filling it. Missing-completely-at-random, missing-at-random, and missing-not-at-random demand different handling, and the *fact* of missingness is often itself a signal (an unfilled income field that correlates with the target). Mean/median imputation fit on the full dataset before the split leaks the test set's distribution, so fit the imputer inside the fold like any other transform. Add a missing-indicator column when absence is informative, prefer model-based or grouped imputation when the data has structure, and never silently drop rows whose missingness is correlated with the outcome — that biases the result and hides it. Record every cleaning decision; a dropped row or a filled value that nobody can explain later is an unaudited assumption.
