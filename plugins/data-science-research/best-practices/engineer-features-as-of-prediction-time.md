# Engineer every feature as of prediction time

A feature is only legitimate if its value would have been knowable at the moment the model makes its prediction. Audit each engineered column against that clock: a "days since last login" computed from a snapshot taken *after* the label's observation window encodes the answer; an aggregate that silently includes the target row leaks the target; a category mean fit on the full dataset leaks the test rows. Fit every fit-based transform (scaler, imputer, target/category encoder) *inside* the cross-validation fold — a scikit-learn `Pipeline` enforces this by construction — and for target-derived encodings prefer out-of-fold or smoothed schemes. When a feature looks too predictive, treat it as a prediction-time-availability question before celebrating it.
