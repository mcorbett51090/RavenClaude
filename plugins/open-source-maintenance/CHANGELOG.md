# Changelog — open-source-maintenance

All notable changes to this plugin are documented here. Format follows [Keep a Changelog](https://keepachangelog.com); this plugin versions per [SemVer](https://semver.org).

## [0.1.0] — 2026-06-23

### Added

- Initial release. The open-source-maintenance team — running a public project people can use, trust, and contribute to.
- **2 agents:** `oss-maintainer-strategist` (license/CLA-DCO, governance & community-health files, issue/PR triage, contributor funnel, bus factor, funding) and `release-and-versioning-engineer` (semver, changelogs, release automation, deprecation/breaking-change windows, coordinated security releases, dependency intake, supply-chain provenance).
- **5 skills:** `choose-an-open-source-license`, `triage-issues-and-prs`, `plan-a-release`, `manage-breaking-changes-and-deprecations`, `coordinate-a-security-release` (shared at the security seam).
- **Knowledge bank (4 docs):** two Mermaid decision trees (licensing + semver/release), `community-health-and-governance.md`, and a dated `oss-tooling-2026.md` category map.
- **8 best-practices**, **4 templates** (CONTRIBUTING, SECURITY, release checklist, governance/maintainer-ladder), and **1 advisory hook** (`flag-oss-hygiene-smells.sh`, POSIX-ERE only).
- Seams to `devops-cicd` (release pipeline), `security-engineering` (vuln analysis), `developer-relations` (community growth), `technical-writing-docs` (docs craft), and `developer-tooling` (monorepo/build).
