# Bake guardrails in as defaults, enforce with policy-as-code

The secure, tagged, compliant configuration should be what you get by default on the paved road, so opting out is the deliberate, owned, harder path. Encode the policy into the template/module and verify off-road usage with policy-as-code (OPA/Conftest/Kyverno) — advisory first, blocking only for the genuinely irreversible. A guardrail enforced by a reviewer's diligence fails the first busy week.
