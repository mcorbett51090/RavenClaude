# Fall back down the locale chain, never to a raw key

A missing `pt-BR` string should render `pt`, then the default locale (usually `en`), not the literal key `home.cta.signup`. Define the fallback chain explicitly — region → language → default — and resolve each lookup down it so a partially-translated locale degrades to readable text instead of exposing internal IDs to a user. A raw key in the UI is both an embarrassing bug and a signal that the catalog drifted from the code. Two corollaries: fall back per *key*, not per *locale* (a locale that's 95% done shouldn't drop to English wholesale because three strings are missing), and make the fallback observable — log or count fallback hits in CI/QA so "silently rendering English" doesn't masquerade as "locale complete." The chain is locale data; read the parent relationships from CLDR's locale inheritance, don't invent them.
