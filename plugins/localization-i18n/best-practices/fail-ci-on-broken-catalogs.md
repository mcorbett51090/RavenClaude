# Fail CI on a broken catalog

A broken catalog should be a red build on the PR that broke it, not a bug a user reports from production. Wire catalog validation into CI as a required check and fail on: missing keys in a required locale, placeholder-set mismatches between source and translation (a dropped `{count}` crashes only that language), broken ICU MessageFormat syntax (unbalanced braces, an undefined plural category, a malformed skeleton), duplicate or orphaned keys, and — for locales declared shippable — incompleteness below threshold. Run the pseudo-locale in the same job so newly hardcoded strings surface here too. The check must be deterministic and offline (validate the catalog files in the repo; don't depend on a live TMS call that can flake). The point is to move every catalog defect left to the cheapest place to fix it — the diff that introduced it — and to make "the build is green" actually mean "the localized build renders." A placeholder-parity failure caught in CI is a one-line fix; the same failure in production is a per-language crash and an incident.
