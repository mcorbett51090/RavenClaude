# Context travels with the string

A bare `"Open"` handed to a translator is a guess: verb or adjective? button or status? imperative or label? The same five letters become different words in most languages, and the translator can't see the screen. Ship context *with* every string into the TMS — a developer comment describing intent, a screenshot or in-context link, a character-limit for constrained UI, a description of each placeholder (`{count}` = number of files, an integer ≥ 0), and the part of speech where it's ambiguous. Most catalog formats carry this natively (gettext `#.` translator comments, XLIFF `<note>`, ARB `@key` metadata, Android `<!-- -->` comments), and a good TMS surfaces screenshots inline. Context is not optional polish: it is the difference between a correct translation and a plausible-looking wrong one that linguistic QA has to catch later at far higher cost. Make the context a required field for any string that isn't a self-evident full sentence.
