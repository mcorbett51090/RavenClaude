# Bidi-isolate every interpolated value

A user name, a file path, a number, or any runtime value spliced into a message can carry its own directionality, and without isolation it reorders the surrounding text in RTL contexts — a Latin filename inside an Arabic sentence scrambles the line, and a phone number can jump to the wrong side. Wrap interpolated values in Unicode bidi isolates (the FSI…PDI pair, `U+2068`…`U+2069`) or use your framework's bidi-isolation helper / the `<bdi>` element so each value is laid out in its own embedding without leaking direction into the sentence. This is not the same as setting `dir=rtl` on the container: the container direction is the paragraph's, isolation is the *value's*. Hand the message format the placeholder; let the renderer isolate it — never let an untrusted runtime string decide where the words around it land.
