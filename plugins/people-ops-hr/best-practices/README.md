# People-ops / HR best-practices

Atomic, enforceable rules the people-ops-hr agents apply. Each file is one rule with a short rationale; the agents cite them by filename. Canonical decision logic lives in [`../knowledge/people-ops-hr-decision-trees.md`](../knowledge/people-ops-hr-decision-trees.md); these rules are the always-on priors.

**Overriding rule:** none of these is legal advice. Where a rule meets employment law, the action is *flag for counsel* — see `flag-employment-law-for-counsel-never-opine`.

| Rule | Gist |
|---|---|
| flag-employment-law-for-counsel-never-opine | Employment-law calls are flagged for counsel, never opined on |
| structure-beats-intuition-in-hiring | Anchored scorecards assessed the same way for every candidate |
| level-before-band | Build the leveling ladder first; the band maps to the level |
| pay-equity-controls-for-legitimate-factors | Control for level/tenure/location/performance; surface the residual |
| compa-ratio-reads-position-not-fairness | Compa-ratio places someone in the range; it is not an equity verdict |
| separate-merit-promotion-and-equity-budgets | Three pools, three purposes; calibrate before communicating |
| handbook-policy-is-plain-language-then-mechanics | Statement -> scope -> rule -> process -> edge cases, consistent |
| hris-is-the-source-of-truth-or-nothing | One canonical record with input controls; payroll depends on it |
| onboarding-is-owned-not-improvised | A named owner + a repeatable list; same good start every time |
| offboarding-is-half-the-lifecycle | Access, final pay, equipment, knowledge transfer, data updates |
| measure-the-funnel-not-the-applicant-count | Conversion + time-in-stage diagnose hiring; volume is vanity |
| candidate-experience-is-the-employer-brand | A fast, fair, communicative process is brand and a conversion lever |
