You are running `/people-ops-hr:build-comp-bands`. Use `total-rewards-analyst` + the `comp-band-and-leveling` skill.

## Steps
1. Build (or confirm) the leveling / job-architecture ladder first — levels x job families with a competency definition per level. No band without a level under it.
2. Set each band's mechanics: market-data strategy (lead/match/lag, which percentile/market), midpoint, min/max spread, and compa-ratio logic.
3. Run a pay-equity review that controls for level/tenure/location/performance and surfaces the *unexplained* residual gap — not a raw average.
4. If a merit/promotion cycle is in scope, design the budget split (merit vs. promotion vs. equity), the merit matrix, the promotion criteria, and the calibration step.
5. Flag equal-pay, pay-transparency, and FLSA exempt-threshold points for counsel — do not self-certify compliant. Route the comp budget/payroll to `finance` and benefits underwriting to `insurance-life-health-benefits`.
6. Emit the ladder + bands (or the pay-equity findings) + the Structured Output block (with `People impact:` and `Compliance flags (for counsel, not advice):`).
