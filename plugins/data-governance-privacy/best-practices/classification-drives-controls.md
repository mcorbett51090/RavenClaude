# Classification is the input to controls, not paperwork

A classification scheme earns its keep only when each level mechanically determines the controls applied — restricted data gets encryption, strict access rules, masking, and retention limits; public data gets none of that — so that tagging an asset automatically provisions its protection. A classification that lives in a spreadsheet and connects to nothing is theater: it tells an auditor you thought about sensitivity while every asset is protected identically regardless of its label. Keep the levels few enough that people actually apply them (a twelve-tier scheme nobody uses governs nothing), and wire each level to the enforcement system that owns the control. The point of saying "this is restricted PII" is that something happens because you said it — if nothing does, you've described the risk without reducing it.
