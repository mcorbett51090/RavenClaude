# Grant data access least-privilege, by role and purpose

Access to classified data should default to none and be granted by role and documented purpose, scoped to the columns and rows the work actually requires — not "everyone in analytics can read everything because it was easier to set up." Over-broad access is the quiet multiplier on every breach and insider risk: the blast radius of a compromised account is exactly the data it could reach, and the standing grants nobody remembers are the ones that show up in the post-incident review. Prefer column- and row-level controls and masking over all-or-nothing table grants, review access on a cadence, and tie every grant to a purpose so it can be revoked when the purpose ends. Least privilege is not friction for its own sake — it is the difference between an incident that exposes one team's data and one that exposes the estate.
