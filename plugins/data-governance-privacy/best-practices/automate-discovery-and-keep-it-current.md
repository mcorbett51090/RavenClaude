# Sensitive-data discovery is continuous, not a one-time scan

PII and sensitive data don't stay where you found them: a new column lands, a developer copies a table into a sandbox, an export job writes a CSV to object storage. So discovery and classification must run continuously — automated scans on a schedule that re-tag as schemas and data move — not a one-time inventory that's stale the week after you finish it. A catalog that was accurate at audit time and ignored since is worse than none, because it gives false confidence: teams trust a map that no longer matches the territory and miss the PII that crept into an untracked location. Wire discovery into the pipeline so new and changed data is scanned as it appears, and treat the catalog as a living system with an owner, not a project that ships and ends. You can't govern what you can't find, and what you found last quarter is not what exists today.
