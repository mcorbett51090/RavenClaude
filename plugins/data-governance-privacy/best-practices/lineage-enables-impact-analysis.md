# Lineage is what makes impact analysis possible

End-to-end lineage — source through every transform to every consumer — is the capability that answers the questions governance actually gets asked: if this source contains PII, which dashboards and exports inherit it; if we delete a person's data here, what downstream copies must also go; if this column is wrong, who was affected. Without lineage, each of those is a manual archaeology dig across systems, and the answer you give is "we think that's all of them" — which is exactly the answer that fails a DSR and a breach assessment. Capture it automatically from pipeline and dbt metadata rather than maintaining a diagram by hand, because hand-drawn lineage rots into fiction the first time a schema changes. Lineage isn't documentation; it's the query engine for "what does this touch," and almost everything else governance does depends on being able to answer that.
