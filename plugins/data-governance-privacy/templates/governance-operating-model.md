# Data governance operating model

- **Ownership:** each domain has an accountable owner + hands-on steward.
- **Classification:** <scheme> applied, highest-risk first.
- **Policies -> controls:** each policy maps to an enforced control (where).
- **Council/RACI:** decisions, exceptions, escalations owned.
- **Maturity roadmap:** inventory -> classify -> control -> monitor (staged).
- **Seams:** enforcement -> data-platform; privacy mechanics -> privacy-compliance-engineer; financial-regulatory -> regulatory-compliance.
