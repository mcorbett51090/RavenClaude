# Data-subject-request runbook

1. **Verify** requester identity.
2. **Locate** all of the subject's data via the catalog/lineage (gap = fix discovery first).
3. **Determine** request type (access / portability / erasure).
4. **Erasure:** check lawful retention obligations (route legal to regulatory-compliance/legal); erase the rest across all systems + propagate.
5. **Access/portability:** export in a portable format.
6. **Verify** completion across systems.
7. **Respond** within the legal deadline; **log** the request + actions.
