# Structure a co-sell motion

You are running `/partnerships-alliances:structure-a-co-sell-motion` for `$ARGUMENTS`. Run it the way the team's specialists would — applying the house opinions in [`../CLAUDE.md`](../CLAUDE.md) §3.

## Steps (traverse top-to-bottom; do not skip)
1. Prove the account overlap — no overlap, no co-sell; say so and stop (readiness gate).
2. Write the joint value proposition — the specific customer outcome neither party delivers alone (§3 #4).
3. Name the reps and the incentive — individual rep pairings + a shared incentive that makes each call (§3 #3).
4. Define the plays by segment — trigger, joint pitch, hand-off, deal-registration step.
5. Pick the first three accounts — each with a rep pairing and a next action.

## Output
A co-sell playbook: account-overlap map, joint value proposition, named rep pairings with a shared incentive, plays by segment, and the first three accounts with owners. See [`../skills/structure-a-co-sell-motion/SKILL.md`](../skills/structure-a-co-sell-motion/SKILL.md), the co-sell-readiness tree in [`../knowledge/partnerships-decision-trees.md`](../knowledge/partnerships-decision-trees.md), and [`../templates/partner-business-plan.md`](../templates/partner-business-plan.md).

## Guardrails
- Apply the §3 house opinions before any method; no unnamed "we'll co-sell" motion.
- No partner PII; cite a source + date for every referral/rev-share figure (or mark it).
- End with owner / date / expected movement on each play.
