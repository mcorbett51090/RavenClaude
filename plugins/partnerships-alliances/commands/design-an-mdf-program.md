# Design an MDF program

You are running `/partnerships-alliances:design-an-mdf-program` for `$ARGUMENTS`. Run it the way the team's specialists would — applying the house opinions in [`../CLAUDE.md`](../CLAUDE.md) §3.

## Steps (traverse top-to-bottom; do not skip)
1. Tie funds to a plan — no MDF without an approved activity plan and an expected sourced-pipeline number (§3 #5).
2. Set eligibility and the funding basis — tier gate + fixed pool / accrual / proposal-based; cite MDF-rate norms with source + date.
3. Require proof of performance — pre-approval, eligible expenses, evidence, claim window.
4. Measure on ROI and set a floor — MDF-sourced-pipeline ROI with a floor below which funds redirect.
5. Report the program ROI — spend, sourced pipeline, blended ROI on the QBR cadence.

## Output
An MDF program: eligibility, funding basis, plan-first approval gate, claim/proof rules, an ROI measure with a floor, and QBR reporting. See [`../skills/design-an-mdf-program/SKILL.md`](../skills/design-an-mdf-program/SKILL.md) and [`../templates/qbr-readout.md`](../templates/qbr-readout.md).

## Guardrails
- Apply the §3 house opinions before any method; no unmeasured MDF.
- MDF-tax treatment is flagged for counsel; cite a source + date for every external rate.
- End with owner / date / expected movement.
