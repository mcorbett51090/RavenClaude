# Use mutation testing to grade the tests themselves

Coverage tells you a line *executed* during the suite; it says nothing about whether any assertion would notice if that line were wrong. Mutation testing closes the gap: it deliberately introduces faults (flip a `>` to `>=`, drop a `!`, return null) and checks whether a test fails — a surviving mutant is a defect your suite would ship. Treat the mutation score as the real measure of test *quality* and run it where it earns its compute: the critical, high-blast-radius modules (payments, auth, pricing), not the whole tree (it's expensive). A surviving mutant is a concrete, actionable assignment — "add the assertion that kills this" — which beats chasing a coverage percentage that rewards executing code without checking it.
