# Put a One-Page Executive Summary With a Recommendation First

**Status:** Pattern
**Domain:** Board / investor reporting
**Applies to:** `finance`

---

## Why this exists

Board members and investors read the summary first and the supporting detail only when something prompts a deeper look. A board pack that buries the "so what" inside 40 slides forces the reader to reverse-engineer the conclusion from the data — and busy readers often don't. The executive summary page serves a second purpose: it forces the finance team to commit to a clear, manageable recommendation or call-to-action before the meeting, not after a 90-minute discussion. A pack without a recommendation is a presentation of data, not a governance instrument.

## How to apply

Every board or investor pack opens with a one-page executive summary that fits on a single slide or page — no exceptions for "complexity."

```
Executive Summary — Required Elements
──────────────────────────────────────
1. HEADLINE  — one sentence: the most important thing the board needs to know this period.
2. PERIOD PERFORMANCE — 3–5 bullets: actuals vs plan / prior period, material moves only.
   Format: "<Metric>: <actual> vs <plan> (<+ / - variance> / <% variance>) — <one-phrase why>"
3. OUTLOOK — 1–3 bullets: forward-looking statement, any revised guidance.
4. RISKS / OPEN ITEMS — 1–3 bullets: the things the board needs to be aware of or act on.
5. RECOMMENDATION / DECISION REQUESTED — explicit: "Board is asked to <approve / note / decide>"
   or "No board action required — for information only."
```

**Do:**
- State the recommendation on the summary page, even if it is "no action needed."
- Keep the summary to one page — if it overflows, cut the performance bullets, not the recommendation.
- Write the summary last (after the supporting deck is built) but place it first in the document.
- Align the headline with the most important narrative in the deck — if the narrative is "cash is tight," the headline says that, not "Q3 results."

**Don't:**
- Open with a table of KPIs and no narrative summary — this is the most common failure mode.
- Write a summary that simply lists topics covered ("this pack covers revenue, expenses, and cash") instead of stating the key conclusion.
- Bury the decision request in a footnote or in the last slide.
- Use passive voice on the recommendation ("it is suggested that") — be direct.

## Edge cases / when the rule does NOT apply

- **Routine lender covenant-compliance certificate** — a structured form with a cover memo; the form governs, but a one-sentence status line ("covenants are in compliance / the following covenant is at risk") still belongs on page one.
- **Monthly management reporting (not board level)** — the discipline applies but the format can be lighter; a one-paragraph header narrative is sufficient.
