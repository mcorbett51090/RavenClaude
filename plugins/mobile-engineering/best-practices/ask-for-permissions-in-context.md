# Request permissions in context, and design for denial

A permission prompt fired at launch, before the user understands why, gets denied — and on both platforms a hard denial is sticky, sending the user to system Settings to undo it. Request push, location, camera, and contacts at the moment the feature needs them, after a short in-app primer that explains the value, so the OS dialog lands when the answer is obviously yes. Then design every feature to work — degraded but functional — when the permission is refused or later revoked; never block core flows on an optional grant. Notification permission in particular is now opt-in everywhere, so treat push as a privilege you earn, handle the "denied" branch as a first-class state, and re-check at use time because the user can change it anytime.
