# Build for native feel, not one UI across both platforms

Users hold each platform's conventions as muscle memory, and an app that ignores them feels foreign even when it functions. Respect platform-native navigation patterns, the system back gesture/button, the expected position of primary actions, native date/share/picker components, haptics, and Dynamic Type / font-scaling so the UI honours the user's accessibility settings. Shipping one pixel-identical design to both stores — an iOS app with Android's affordances or vice versa — saves design effort and spends user trust. Cross-platform frameworks make this harder, not easier: shared code is fine, but the navigation, the controls, and the gestures should still read as the platform the user chose. Match the platform where it's felt; share the logic underneath.
