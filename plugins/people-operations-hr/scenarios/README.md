# People-Ops scenarios bank

Dated, scope-tagged, **unverified** engagement narratives for the `people-operations-hr` plugin (the marketplace scenarios pattern; see [`../../ravenclaude-core/skills/scenario-retrieval/SKILL.md`](../../ravenclaude-core/skills/scenario-retrieval/SKILL.md)).

**How to use these (the standing caveat):** a scenario is a *secondary* source. Surface a matching one only behind the mandatory unverified-scenario preamble, and never let it override the cited [`../knowledge/`](../knowledge/) bank or qualified counsel (CLAUDE.md §2). Every scenario carries **no employee PII** (§2). Benchmark figures inside a scenario are `[unverified — training knowledge]` unless a dated source is attached — validate against the client's own HRIS/ATS data before any deliverable (§3 #8).

Each file ends with an **"Action for the next consultant"** lesson — the transferable move, not the client specifics.
