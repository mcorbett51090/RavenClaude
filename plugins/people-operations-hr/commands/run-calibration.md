You are running `/people-operations-hr:run-calibration`. Use the
`performance-and-comp-analyst` discipline and the `performance-and-calibration` skill.

## Steps

1. **Confirm cycle context.** Extract from the argument (or ask): org scope, headcount, rating
   scale in use, cycle timeline, calibration model (pre vs post rating), and any known
   distribution expectations (forced distribution vs guided distribution).

2. **Traverse the performance-and-calibration skill.** Read
   [`../skills/performance-and-calibration/SKILL.md`](../skills/performance-and-calibration/SKILL.md),
   specifically §4 (pre-work packet) and §5 (calibration facilitation).

3. **Build the pre-work packet.** Design the manager submission template:
   - Proposed rating per direct report.
   - 2–3 sentence evidence summary per person.
   - Promotion nominees with evidence.
   - Anyone on a formal performance plan (PIP).
   Produce a calibration deck format (one row per employee: proposed rating, evidence summary,
   prior cycle rating, tenure in role).

4. **Write the facilitation guide.** Produce a session agenda:
   - Opening framing (purpose is calibration, not consensus).
   - Distribution review (flag outlier managers).
   - Employee-by-employee discussion protocol.
   - Anti-bias prompts: recency bias, halo effect, manager-advocacy skew.
   - Documentation protocol for recalibrated ratings.

5. **Produce the post-session output template.** Finalized ratings table, promotion list with
   evidence, PIP-tracking list, recalibration log (who changed, why, by how much).

6. **Design the communication plan.** Manager guidance for rating conversations: lead with
   evidence, not the number; separate timing from merit conversation (≥1 week gap).

7. **Emit the Structured Output block.** Include: org scope, rating scale, calibration model,
   pre-work packet design, facilitation guide summary, and handoff recommendations
   (performance-and-comp-analyst for merit cycle; people-analytics-engineer for rating
   distribution analytics).
