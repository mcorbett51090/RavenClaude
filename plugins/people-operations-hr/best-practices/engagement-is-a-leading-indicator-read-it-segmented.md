# Engagement is a leading indicator — read it segmented

**Status:** Pattern. **Constitution:** §3 #4, §4.

## Use when
Any People-Ops / HR deliverable where this question is in play — read, applied, and cited whole.

## The rule
A company-wide eNPS hides the team-level, tenure-cohort, and manager problems that actually drive regretted exits; read it segmented.

## Why it matters
This is house opinion §3 #4, distilled into a citable rule. People leaders act on these deliverables, so a framing error here is not academic — it sends real headcount, comp, or retention decisions the wrong way. The rule is cheap to apply and expensive to skip.

## How to apply
- Apply this **before** reaching for a method — it sets the framing, not the conclusion.
- Make every number in the deliverable carry a definition, a window, and a baseline (§3 #1).
- Cite a source + date for any external benchmark, or mark it `[unverified — training knowledge]` (§3 #8).
- When this rule and another both apply, route to [`people-ops-lead`](../agents/people-ops-lead.md) to sequence them — overlapping signals usually mean two drivers at once.
- Keep employee PII out of the deliverable; route legal/regulatory determinations to qualified counsel (§2).
- Close with a recommendation that has an owner, a date, and an expected metric movement.

## The anti-pattern this prevents
The §4 failure mode: acting as if “engagement is a leading indicator — read it segmented” weren't true — the most common way a People-Ops analysis quietly misleads the leader who acts on it. The plugin's advisory hook flags a deliverable that reads as if this rule were ignored.

## Worked example
_Situation:_ A company-wide eNPS hides the team-level, tenure-cohort, and manager problems that actually drive regretted exits; read it segmented. _Wrong move:_ skipping straight to a fix without applying this lens — you treat a symptom as the cause and prescribe the expensive wrong remedy. _Right move:_ apply the rule first; the framing it forces usually changes which of the two likeliest causes you chase, and it always changes what evidence you ask for next.
