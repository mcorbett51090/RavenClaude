# The build belongs to the system layer — specify the contract, hand off the build

This plugin defines the funnel, the metric, the routing intent, and the comp model; the Salesforce objects/flows/Apex/validation rules are `salesforce`, the warehouse revenue mart is `data-platform`, and the dashboard is `tableau`. RevOps owning the platform build is the recurring failure mode — it produces a half-built CRM nobody maintains and a metric defined inside a dashboard instead of in one source of truth. The seam is "could this change a consumer's CRM platform / warehouse / BI tool" — if yes, it's a hand-off with a written contract (the data model, the required fields, the metric definition, the automation intent), not a thing this layer builds. Specify the contract precisely enough that the system layer can build it without re-litigating the definition, name what you're handing to whom, and keep the single source of revenue truth in the model — never let the metric drift into the dashboard or the spreadsheet. RevOps is the revenue engine; those plugins are the systems it runs on.
