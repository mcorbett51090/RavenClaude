# Data quality is the substrate, not a cleanup project

Duplicates, missing fields, and dead accounts silently corrupt every downstream metric — coverage, win-rate, attribution, comp attainment — so enforce quality at the point of entry, not in a quarterly scrub. A quarterly cleanup is a treadmill: the bad data is already in the forecast by the time you catch it, and the same gaps reappear next quarter because nothing stopped them entering. Require the fields a metric depends on as validation at creation, dedupe on entry (matching rules + a merge path), and put an owner on every account so "dead account" is a state someone changes, not a guess. The build of those validation rules and dedupe automation is a `salesforce` hand-off; this plugin specifies *which* fields are required *because a metric depends on them* and *what a duplicate means* — the system layer enforces it. Treat data quality as the floor the whole revenue engine stands on, because a precise number computed on garbage is still garbage.
