# Routing and scoring are SLAs, not suggestions

Every lead has a defined owner and a speed-to-lead clock, and every lead score is a documented model with a feedback loop — not a static point table nobody revisits. An unrouted lead is lost revenue; speed-to-lead is often the single highest-leverage GTM fix in an org, so put a clock on assignment with an accept/reject + re-route loop. A lead score that is never validated against actual conversion is astrology — the feedback loop that revisits the weights against what actually closed *is* the model, not the point table.
