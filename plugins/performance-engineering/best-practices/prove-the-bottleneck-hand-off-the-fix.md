# Prove the bottleneck, hand off the fix

This layer localizes the constraint; it does not fix it in place. When profiling names a slow query, the fix is an index or a rewrite that **`database-engineering`** owns. When the constraint is a client paint (LCP, bundle, render path), it routes to **`frontend-engineering`**. When it's a dependency that needs a retry/bulkhead/circuit-breaker, that's **`backend-engineering`**. When the gap is a customer SLO or error budget, that's **`observability-sre`**. Tuning the query or rewriting the front-end *here* duplicates another team's craft, drifts the domain boundary, and skips their review. Do the performance work — model the workload, run the test, read the flame graph, compute the capacity — then return a named, evidenced constraint and the seam it routes to. Prove the number; let the owner move it.
