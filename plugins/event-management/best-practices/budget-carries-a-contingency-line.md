# The budget carries a contingency line

**Status:** Absolute rule
**Domain:** Event strategy / budget
**Applies to:** `event-management`

---

## Why this exists

Events surprise you: AV overages, F&B guarantees that come due, attrition, a last-minute shuttle. A budget with no buffer breaks on the first surprise, and the overrun comes out of margin or scope mid-event when you have the least flexibility. A named contingency line absorbs the variance you know is coming.

## How to apply

- Add an explicit **contingency / buffer** line to the budget — commonly **10-20%** of total, higher for first-time or hybrid/complex events.
- Make it a real line item, not a vague "we'll find it" — it has a number and is reserved.
- Track contingency spend post-event so next time's buffer is calibrated to reality.

**Do:** size the contingency to the event's risk; keep it as a named line.
**Don't:** ship a budget that sums to exactly the expected cost with no buffer.

## Edge cases / when the rule does NOT apply

A tiny, fully-refundable, low-commitment event may carry a token buffer — but the line still exists, even if small.
