# Post-event debrief while it is fresh

**Status:** Pattern
**Domain:** Event measurement / continuous improvement
**Applies to:** `event-management`

---

## Why this exists

The detail that makes a debrief useful — which transition ran long, which vendor under-delivered, which funnel stage leaked, what the plan B caught — evaporates within days. A debrief held "next quarter" produces vague generalities. Run it while the team still remembers the day and the lessons carry forward to the next event.

## How to apply

Within days of the event, capture:

- **Funnel actuals** vs target (reach → register → attend), including no-show rate.
- **Budget actual vs plan**, including where the contingency was spent.
- **What broke**, what the plan B caught, and what had no plan B.
- **Sponsor fulfillment** status and renewal signal.
- **3-5 carry-forward lessons** for the next event.

**Do:** schedule the debrief *before* the event, on the calendar for a few days after.
**Don't:** let the team disperse without capturing the perishable detail.

## Edge cases / when the rule does NOT apply

A one-off event with no successor still debriefs — the lessons transfer to adjacent events and to the organization's playbook.
