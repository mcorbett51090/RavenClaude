# Measure against the goal you set

**Status:** Absolute rule
**Domain:** Event measurement / ROI
**Applies to:** `event-management`

---

## Why this exists

After an event there's a strong pull to report whatever number looks good — total attendance, badge scans, social impressions. But the event was funded for a *specific* goal with up-front KPIs. Reporting a vanity metric instead of the chosen KPI hides whether the event actually worked and makes the next budget decision blind. The metric was chosen for a reason; honor it.

## How to apply

| Goal | Measure | Not |
|---|---|---|
| Revenue | Net revenue vs fully-loaded cost | Gross ticket sales |
| Pipeline | Qualified leads / pipeline value / cost per lead | Badge scans |
| Community | NPS / return-intent / engagement | Raw attendance |
| Education | Completion / assessment lift | Sign-ups |

**Do:** compute ROI as outcome ÷ **fully-loaded cost**, against the up-front KPI.
**Don't:** swap the success metric after the fact; don't report attendance as if it were the goal.

## Edge cases / when the rule does NOT apply

Reporting secondary metrics *alongside* the primary KPI is fine and useful — the rule is that the primary goal's KPI is always reported, not replaced.
