# Name the go/no-go criteria early

**Status:** Absolute rule
**Domain:** Event strategy / risk
**Applies to:** `event-management`

---

## Why this exists

By the time an event "feels" like it's failing, the non-refundable spend is usually already committed — venue deposits, AV contracts, catering guarantees. The only cheap moment to cancel or pivot is *before* that spend ramps, and you can only act on it then if you decided the threshold in advance. A go/no-go named after the fact is a sunk-cost trap.

## How to apply

- Set **gate(s) with a date and a hard threshold**: registrations sold, sponsorship secured, speakers/key vendors confirmed.
- Derive the registration threshold from **break-even**, not optimism.
- Define what each outcome triggers: **GO**, **pivot** (cut scope / change format), or **no-go** (cancel).
- Place the gate before the largest non-refundable commitment.

**Do:** tie every gate to a calendar date and a number.
**Don't:** carry a vague "we'll see how sales go" with no threshold or date.

## Edge cases / when the rule does NOT apply

A recurring, fully-de-risked event with reliable demand may run a lightweight gate — but the date and threshold still exist on paper.
