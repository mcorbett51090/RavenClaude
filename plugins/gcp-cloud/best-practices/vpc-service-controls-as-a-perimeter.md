# Use VPC Service Controls to draw an exfiltration perimeter

IAM answers 'who may call this API'; it does nothing to stop a credential that has been phished or leaked from pulling your data out to an attacker's project. VPC Service Controls draws a service perimeter around projects so that even a valid identity can't move data from BigQuery, GCS, or other managed APIs across the boundary — turning a leaked key from a full data breach into a contained, logged, denied call. Pair it with Private Service Connect / Private Google Access so the managed services are reachable only from inside the perimeter. Run the perimeter in dry-run mode first to find legitimate cross-boundary flows, then enforce. For data of real sensitivity, IAM is the lock and VPC-SC is the wall around the building.
