# Use CMEK where you need key control, not encryption-by-default

Google encrypts data at rest by default, so reach for Customer-Managed Encryption Keys (Cloud KMS) when you specifically need control the default doesn't give: a key you can rotate on your schedule, an audit trail of every decrypt, the ability to revoke access by disabling the key, or a compliance mandate for customer-held keys. CMEK is the right call for regulated or sensitive data; it is needless operational overhead for an internal scratch dataset. When you do use it, manage key location and rotation deliberately, watch for the dependency it creates — destroying or disabling the key makes the data unrecoverable — and enforce CMEK on the services that require it via org policy so a new resource can't slip in with default keys. Match the control to the data's actual sensitivity.
