# Specify the requirement, hand off the build

This plugin owns the content plan, the SEO program, and the lifecycle system — not the site, the test engine, or the warehouse. When work crosses into the build/measurement layer, return the content-and-growth slice with an explicit, buildable requirement and name the owner: the marketing-site build and render strategy (SSR/SSG for crawlability) → `web-design`; the A/B / multivariate test of a headline, CTA, subject line, or flow → `experimentation-growth-engineering`; attribution, identity, and the organic-to-pipeline pipeline → `data-platform`. Don't build it here and don't leave the seam implicit — a deliverable that buries the hand-off ships a half-spec nobody can execute. Specify what you need, name who builds it, make the seam legible in every report (the `Handoff to build/measurement:` line).
