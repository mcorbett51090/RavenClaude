# Search intent is the unit, not the keyword

A query is a window into a job. Cluster queries by the intent behind them (informational / commercial / transactional / navigational) and map each cluster to the single page that best serves it — a transactional query pointed at a blog post loses to a product page every time. Volume without intent-fit is vanity, and three thin pages competing for one query (cannibalization) beat each other; consolidate them into one pillar. Match the page to the intent before any on-page tuning.
