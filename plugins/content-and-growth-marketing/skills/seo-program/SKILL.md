# SEO Program

## Search intent is the unit
Cluster queries by the intent behind them (informational / commercial / transactional / navigational) and map each cluster to the single page that serves it. Match the page to the intent — a transactional query on a blog post loses to a product page. Score opportunity as volume × intent-fit × difficulty, and name the content gaps.

## Technical foundation first
A great page that can't be crawled or indexed ranks for nothing. Confirm crawlability, indexation, and site architecture; check Core Web Vitals and structured data. Optimizing an unindexed page is motion, not progress — fix indexation before on-page.

## On-page + internal linking as designed assets
Titles, headings, entity/semantic coverage, and the internal-link topology (pillar ↔ cluster) are deliberate structure, not whatever links happened to get added. Consolidate cannibalizing thin pages into one canonical pillar per intent cluster.

## SERP features and AEO/GEO
Target featured snippets, People-Also-Ask, and knowledge panels with answer-shaped content + structured data. Treat AEO/GEO (AI Overviews, LLM answers) as a first-class surface: citable, entity-rich, Q&A-structured content — measured distinctly from blue-link rank.

## SEO is a program, not an audit
Deliver a prioritized, owned, recurring backlog — not a one-time PDF that ages out. Don't quote a search volume, difficulty, or ranking you didn't pull this session; cite the tool + date or flag it `[verify-at-build]`.

## Output
A keyword + search-intent map, a prioritized technical + on-page + AEO/GEO diagnosis, ordered by impact. Route content gaps to `content-strategist`, the site/CWV build to `web-design`, and ranking/traffic analytics to `data-platform`.
