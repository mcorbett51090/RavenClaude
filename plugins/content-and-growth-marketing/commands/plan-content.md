You are running `/content-and-growth-marketing:plan-content`. Use `content-strategist` + the `content-strategy-and-briefs` skill.

## Steps
1. Map the audience + jobs-to-be-done by funnel stage. If no shared, repeated job is named, say so before proposing a calendar.
2. Design topic clusters — the 2-3 pillar themes, the pillar pages, the cluster content under each, and the internal-link topology (route SEO validation to `seo-program-lead`).
3. Set a sustainable editorial cadence sized to real capacity; list explicit non-goals (topics/formats the program will NOT chase).
4. For any single piece, write a complete brief: audience + job, search intent + target query, angle/POV, outline, internal links in/out, CTA, success metric.
5. Plan distribution + repurposing — the atomization (one pillar → social/email/video/snippets) and the channel-fit map.
6. Route the builds: site/templates → web-design; SEO targets → seo-program-lead; lifecycle wiring → lifecycle-marketing-engineer; analytics → data-platform.
7. Emit the topic-cluster strategy and/or the content brief + the Structured Output block (with `Audience job served:` and `Handoff to build/measurement:`).
