You are running `/content-and-growth-marketing:audit-seo`. Use `seo-program-lead` + the `seo-program` skill.

## Steps
1. Research keywords + cluster by search intent (informational / commercial / transactional / navigational); map each cluster to an existing or net-new page. Cite the tool + date for any figure, or flag it `[verify-at-build]`.
2. Diagnose the technical foundation: crawlability, indexation, site architecture, Core Web Vitals, structured data. Indexation before optimization.
3. Tune on-page + internal linking (pillar ↔ cluster); consolidate any keyword-cannibalizing thin pages into one canonical pillar.
4. Target SERP features (snippets, PAA, knowledge panels) and build an AEO/GEO plan (answer-shaped, citable, Q&A-structured content) — measured distinctly from classic rank.
5. Order everything by impact into a prioritized, owned backlog (a program, not a one-time PDF). Name the content gaps.
6. Route the builds: content gaps → content-strategist; site/CWV → web-design; ranking/traffic analytics → data-platform.
7. Emit the keyword/intent map + the prioritized diagnosis + the Structured Output block (with `Search intent served:` and `Handoff to build/measurement:`).
