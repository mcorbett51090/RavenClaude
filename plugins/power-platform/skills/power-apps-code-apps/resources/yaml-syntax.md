# Power Apps YAML Source Format (.pa.yaml) Reference

Used for Canvas App source representation and migration tasks.

## Overview

The `.pa.yaml` format (v3.0) is the source code representation of Power Apps canvas apps.
It replaces the retired `.fx.yaml` format. These files are auto-generated by Power Apps
Studio and are primarily read-only.

## Schema Structure (v3.0)

### Top-Level Properties

```yaml
App:
  Properties:
    # Power Fx formulas for app-level settings

Screens:
  HomeScreen:
    Properties:
      # Screen-level Power Fx formulas
    Children:
      - MyButton:
          Control: Button
          Properties:
            Text: ="Click Me"
            OnSelect: =Navigate(DetailScreen)

ComponentDefinitions:
  MyComponent:
    # Reusable component definition

DataSources:
  Contacts:
    Type: Table
    ConnectorId: shared_commondataservice
    Parameters: {}

EditorState:
  ScreensOrder:
    - HomeScreen
    - DetailScreen
  ComponentDefinitionsOrder:
    - MyComponent
```

### Control Instances

Every control requires a `Control` property specifying the ControlTypeId:

```yaml
MyButton:
  Control: Button
  Properties:
    Text: ="Submit"
    OnSelect: =SubmitForm(Form1)
    Fill: =RGBA(0, 120, 212, 1)
```

**ControlTypeId pattern:** `^([A-Z][a-zA-Z0-9]*/)?[A-Z][a-zA-Z0-9]*(@\d+\.\d+\.\d+)?$`
- Starts with uppercase letter
- Optional namespace prefix (e.g., `MyLib/Button`)
- Optional version suffix (e.g., `@1.0.0`)

**Disallowed ControlTypeIds:** `AppInfo`, `HostControl`, `Screen`, `AppTest`, `TestCase`, `TestSuite`

**Not yet supported:** `CommandComponent`, `DataComponent`, `FunctionComponent`

### Power Fx Formula Values

All property values use the `pfx-formula` type:
- String starting with `=` is a Power Fx expression: `="Hello " & User().FullName`
- `null` means use default value
- Only properties that differ from defaults are serialized

### Component Custom Properties

```yaml
ComponentDefinitions:
  MyInput:
    CustomProperties:
      - Name: Value
        PropertyKind: Input
        DataType: Text
        Default: =""
      - Name: OnChange
        PropertyKind: Event
        ReturnType: Boolean
        Parameters:
          - Name: NewValue
            DataType: Text
            IsOptional: false
```

**PropertyKind values:** `Input`, `Output`, `InputFunction`, `OutputFunction`, `Event`, `Action`

**DataType values:** `Text`, `Number`, `Boolean`, `DateAndTime`, `Screen`, `Record`,
`Table`, `Image`, `VideoOrAudio`, `Color`, `Currency`

### ZIndex and Ordering

- Controls use ascending ZIndex starting from 1
- Order in `Children` array determines stacking
- Normal controls: ascending order
- Responsive layout controls: descending order

## Key Constraints

1. Files are auto-generated by Power Apps Studio -- **read-only**
2. External editing only supported through Power Platform Git Integration
3. Manual edits outside Git Integration may be **silently lost**
4. Merging and conflict resolution only work within Git Integration
5. Legacy `.fx.yaml` cannot be directly converted -- must be re-imported and re-exported

## Common Power Fx Patterns (for Migration Reference)

### Navigation
```
Navigate(ScreenName)
Navigate(ScreenName, ScreenTransition.Fade)
Back()
```

### Data Operations
```
LookUp(Contacts, ID = selectedId)
Filter(Contacts, Status = "Active")
Sort(Contacts, FullName, SortOrder.Ascending)
```

### String Operations
```
"Hello " & User().FullName
Len(TextInput1.Text)
Right(Text, 5)
Substitute(Text, "old", "new")
```

### Conditional
```
If(condition, trueValue, falseValue)
Switch(value, match1, result1, match2, result2, defaultResult)
```

### Collections
```
Collect(MyCollection, {Name: "Item"})
ClearCollect(MyCollection, sourceTable)
Remove(MyCollection, record)
```

## Migration: Canvas App to Code App

When porting a Canvas App to a Code App:

1. **Unpack the .msapp:**
   ```bash
   pac canvas unpack --msapp ./MyApp.msapp --sources ./canvas-src
   ```

2. **Read the .pa.yaml files** to understand:
   - Screen structure and navigation
   - Data sources and connections
   - Power Fx formulas (business logic)
   - Component hierarchy

3. **Map to Code App equivalents:**
   | Canvas (Power Fx) | Code App (TypeScript) |
   |---|---|
   | `Navigate(Screen)` | `react-router navigate("/screen")` |
   | `LookUp(Table, cond)` | `Service.get(id)` |
   | `Filter(Table, cond)` | `Service.getAll({ filter: "..." })` |
   | `Collect(Coll, item)` | `useState` or Zustand store |
   | `Set(variable, val)` | `useState` or Zustand store |
   | `User().FullName` | `getContext().user.fullName` |
   | `RGBA(r,g,b,a)` | CSS `rgba(r,g,b,a)` or Tailwind |
   | `If(cond, a, b)` | Ternary `cond ? a : b` |
   | `SubmitForm(Form1)` | `Service.create(data)` or `Service.update(id, data)` |

4. **Recreate UI** using React components with Tailwind CSS or Radix UI
