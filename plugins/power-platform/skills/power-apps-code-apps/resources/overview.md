# Power Apps Code Apps -- Overview Reference

## What Are Code Apps?

Code Apps are a **preview feature** in Power Apps that enable developers to build custom
web applications using standard open-source web technologies (React, Vue, TypeScript)
within VS Code, while running as first-class citizens on the Power Platform.

Unlike Canvas Apps (low-code, Power Fx) or PCF Controls (components embedded in forms),
Code Apps are **standalone applications** with full DOM control that inherit the Power
Platform's authentication, connectivity, and governance.

## Enabling Code Apps (Admin Setup)

1. Navigate to Power Platform admin center (`https://admin.powerplatform.microsoft.com`)
2. Go to **Manage > Environments > [Select environment]**
3. Navigate to **Settings > Product > Features**
4. Toggle **"Enable code apps"**
5. Select **Save**

## Prerequisites

- Node.js LTS
- Git
- Visual Studio Code
- Power Platform CLI (`pac`) version 1.51.1+
- Power Apps Premium license (for end-users)

## Useful URL Parameters

- `hideNavBar=true` -- Hides the Power Apps navigation bar
  - Example: `https://apps.powerapps.com/play/e/{envId}/a/{appId}?hideNavBar=true`

## Key Capabilities

- **Full web framework support**: React, Vue, and other popular frameworks
- **1,500+ connectors**: Access Power Platform connectors from JavaScript/TypeScript
  via auto-generated typed service classes
- **Entra ID authentication**: Built-in, managed by the host. No auth code needed.
- **Connector consent**: Users get a consent dialog for connector permissions.
  Admin consent suppression available for Microsoft OAuth and custom Entra ID connectors.
- **Platform governance**: App quarantine, DLP (enforced at launch), conditional access
  per app, tenant isolation, Azure B2B external user access, health metrics in admin center.

## Architecture Layers

| Layer | Component | Responsibility |
|-------|-----------|---------------|
| 1 | Your Code | Application logic calling SDK APIs |
| 2 | Power Apps SDK | Public APIs, generated models/services, connector data requests |
| 3 | Power Apps Host | End-user auth, app loading/initialization, error handling |

## Four Core Components

1. **Power Apps SDK** (`@microsoft/power-apps` npm package) -- APIs for app code
2. **Generated Models & Services** -- TypeScript files auto-generated by CLI for each data source
3. **power.config.json** -- Metadata config file managed by CLI (do not edit manually)
4. **Power Apps Host** -- Runtime environment that loads and runs the app

## Code Apps vs. Canvas Apps vs. PCF

| Feature | Canvas Apps | PCF Controls | Code Apps |
|---------|-------------|-------------|-----------|
| Development | Low-code / Power Fx | TypeScript / React | TypeScript / React / Vue |
| Host | Proprietary Player | Embedded in Form | Standalone Player |
| File Format | .msapp (Binary/YAML) | Web Standard | Web Standard |
| Data Access | Connectors (implicit) | context.webAPI | Generated services via SDK |
| Auth | Implicit | Implicit | Implicit (via Host) |
| Build Tool | Cloud Studio | msbuild / pac pcf | Vite / Rollup / pac code |
| Version Control | Limited (.pa.yaml) | Standard Git | Standard Git |
| DOM Control | None | Control-scoped | Full page |

## The "PCF Trap" -- Common Mistake

Do NOT assume Code Apps work like PCF controls. Key differences:
- PCF receives a `context` object passed from the form host
- Code Apps must call `getContext()` from the SDK
- PCF uses `context.webAPI.retrieveMultipleRecords()`
- Code Apps use generated service classes (e.g., `ContactService.getAll()`)
- PCF has `context.parameters` -- Code Apps do NOT

## Browser Compatibility Note

Starting December 2025, Chrome and Edge block public origins from accessing local
network endpoints by default. For local development:
- Grant browser permissions when prompted
- Or configure enterprise policies
- For embedded scenarios, add `allow="local-network-access"` to iframe tags
