# Code what was documented, not the reverse

**Status:** Absolute rule.

**Rule:** Coding follows the documented, medically necessary care delivered. Working backward from a
target reimbursement to the code that produces it is fraud risk. The documentation and the medical
necessity come first; the code reflects them.

## Why

The compliant direction of travel is care → documentation → code. When a target reimbursement drives
the code instead, the clinic is billing for a service shaped by the desired payment rather than the
care actually delivered — the definition of upcoding and a False Claims exposure. The discipline is
not merely ethical; a code that the documentation doesn't support is also the most common denial and
the first thing an audit unwinds.

## What it looks like in practice

- If documentation doesn't support a code, the fix is the documentation (or the care plan), never a
  different code chosen to hit a number.
- Productivity and revenue targets are pursued through adherence, denial prevention, and accurate
  coding — not through units the care doesn't justify.
- Unit counts trace to documented timed minutes; CPT selection traces to documented skilled,
  necessary service.

## Anti-pattern

Setting a per-visit revenue or unit target and selecting codes/units to meet it. The hook flags
billing/coding decisions stated with no medical-necessity or documentation basis.
