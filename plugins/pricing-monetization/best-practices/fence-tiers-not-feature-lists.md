# Fence tiers, don't list features

Good-better-best works only when each tier is fenced by a dimension the customer
*self-selects* on — scale, use case, support, security/compliance. Tiers separated by
a longer feature list train customers to wait for the cheap plan to grow the feature,
and give the sales team nothing to anchor on.

**Do:** name the fence for every tier boundary; put long-tail features in add-ons, not
a fourth tier.

**Flag:** any tier ladder where the only difference between plans is the number of
features.
