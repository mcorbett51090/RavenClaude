# Single-instance the app and validate deep-link payloads

Most desktop apps should run as a single instance: acquire a single-instance lock at startup, and when a second launch happens (the user double-clicks an associated file, or the OS hands off a `myapp://` deep link), focus the already-running window and pass it the new argument rather than spawning a duplicate process with its own state. Register custom URL schemes and file associations through this single-instance layer so the app reacts whether it was already running or cold-started. Critically, the file path or deep-link URL is **untrusted input** — it can be crafted by a web page or another app — so validate its shape and authorize the action before acting on it; a deep link that directly triggers a privileged operation without validation is an attack vector, not a feature.
