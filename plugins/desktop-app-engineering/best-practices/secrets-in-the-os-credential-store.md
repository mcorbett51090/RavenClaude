# Store secrets in the OS credential store

Tokens, refresh tokens, and other secrets go in the operating system's credential store — Keychain (macOS), Credential Manager (Windows), or libsecret/Secret Service (Linux) — accessed via Electron `safeStorage` or Tauri's Stronghold/keyring, never in a plaintext `config.json`, `localStorage`, or a file beside the executable. On-device storage is on a device you don't control: a plaintext token is readable by any other process running as the user, by backup tooling, and by anyone who gets the disk. When a Linux system has no Secret Service available, name the weaker fallback explicitly (e.g. an encrypted file with a stated, weaker guarantee) and surface it to the user — do not silently downgrade to plaintext. The OAuth/token flow that produces these secrets is owned by `auth-identity`; this team owns where they land.
