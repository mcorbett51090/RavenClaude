# RFI & Submittal Workflow

## Write an answerable RFI
State the conflict with drawing/spec references, ask **one** specific question, propose a resolution, flag the cost/schedule impact, and give a needed-by date. "Please advise" or five bundled questions burns a review cycle. Log it: number, date sent, ball-in-court, needed-by, date returned, disposition. A change-bearing answer routes to `cost-and-change-controls-lead` before it's built.

## Submittal register (scheduled backward)
Key submittals to the spec sections. Compute required-by = install date − lead time − review time. If the required-by is already in the past, the item is late — flag it now. Track ball-in-court and the disposition (approved / approved-as-noted / revise-and-resubmit); "approved as noted" still needs the noted changes incorporated before close.

## Document control
Maintain the current revision per sheet plus the ASIs/bulletins/supplemental instructions that supersede it, and a transmittal log. The "are we building off the current set" check catches a superseded sheet on the wall before it becomes rework.

## Daily logs & minutes
A daily log is contemporaneous — weather, crews/headcount, work performed, deliveries, delays, visitors, written the same day. Minutes are owned, dated action items, not a transcript.

## Output
A logged RFI tracked to a dated response, a submittal register with back-calculated required-by dates and ball-in-court, a reconciled current set, or action-item minutes. Route the design answer to `architecture-aec`, a change to `cost-and-change-controls-lead`, the schedule build to `project-management`.
