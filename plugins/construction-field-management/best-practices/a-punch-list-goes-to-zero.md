# A punch list goes to zero

Each punch item carries a responsible trade, a location, a description, and a date — and the list is driven to zero, not parked. Substantial completion is not final completion: a punch list that lingers is open scope, a withheld-money argument, and a closeout blocker rolled into one. Track items to closed-and-verified (re-inspected, not just claimed done), sort by trade so each sub gets one actionable list, and report the count remaining against a target date. An item with no responsible trade or no date can't be closed, so it never is.
