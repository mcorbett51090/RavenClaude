# Construction-field-management best-practices

Atomic, enforceable rules the construction-field-management agents apply. Each file is one rule with a short rationale; the agents cite them by filename. Canonical decision logic lives in [`../knowledge/construction-field-management-decision-trees.md`](../knowledge/construction-field-management-decision-trees.md); these rules are the always-on priors.

| Rule | Gist |
|---|---|
| one-question-rfi-tracked-to-a-dated-response | An RFI asks one answerable question and is logged to a dated response |
| ball-in-court-is-the-unit-of-progress | Every open item has one owner and a due date, or it's a stall |
| nothing-gets-built-unpriced-or-uninspected | Price changes before building; verify quality before cover-up |
| schedule-submittals-backward-from-the-install | Required-by = need-by − lead time − review time |
| field-builds-off-one-current-set | Document control: current revision + the ASIs/bulletins that supersede it |
| cost-report-needs-a-cost-to-complete | Budget-vs-actual is a mirage without committed/forecast and CTC |
| safety-is-planned-per-task-osha-is-the-floor | A JHA names hazard + control per step; the talk matches today's risk |
| a-punch-list-goes-to-zero | Each item has a trade, a location, a date; the list closes, it doesn't linger |
| closeout-is-a-package-that-releases-retainage | O&M, as-builts, warranties, CO assembled — or the owner holds the money |
| contemporaneous-records-win-disputes | Logs are written when the event happens — the project's memory and its evidence |
| route-design-intent-to-an-rfi-not-the-field | Field ambiguity → an RFI to design, not a superintendent's interpretation |
| an-sov-is-the-billing-contract-not-a-front-load | SOV ties to cost codes; modest front-loading is normal, abusive gets rejected |
