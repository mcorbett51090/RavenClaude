# IC Memo Must Lead with the Deal Thesis, Not the Property Description

**Status:** Pattern
**Domain:** Commercial real estate
**Applies to:** `commercial-real-estate`

---

## Why this exists

An IC memo that opens with "The property is a 120,000 sf Class B office building located in Denver's Cherry Creek submarket, built in 1998..." has buried the investment decision in a property brochure. The investment committee's job is to decide whether to commit capital; they need to see the thesis, the return, and the risk — in that order — before the property description. A memo that makes the committee read four pages before reaching the recommendation is poorly structured, and poorly structured memos produce imprecise decisions.

## How to apply

Lead every IC memo with a structured executive summary that answers three questions on the first page:

```
IC Memo — Executive Summary Structure
────────────────────────────────────────
Deal name / property:   ________________
Date:                   ________________
Analyst:                ________________

1. THE THESIS (2–3 sentences)
   What is the investment strategy, and why does this asset execute it?
   Example: "Acquire a 90%-leased suburban industrial property in a supply-constrained
   submarket at a 200 bps spread over the 10-yr Treasury. Hold 5 years; NOI grows
   via two scheduled rent bumps (3% annual) and a single 2027 re-leasing event
   at market. Sell at a 25 bps cap expansion."

2. THE RETURNS (one table)
   | Metric | Going-in | Stabilized | Exit |
   |--------|----------|------------|------|
   | Cap rate  |   |   |   |
   | Levered IRR (base) |   |   |   |
   | Levered IRR (downside) |   |   |   |
   | MOIC |   |   |   |
   | DSCR Year 1 |   |   |   |

3. THE RISK / THE BEAR CASE (bulleted)
   - [Primary risk]: ________________  Mitigant: ________________
   - [Secondary risk]: ________________  Mitigant: ________________
   - [Downside scenario]: Exit cap expands to ___%; levered IRR falls to ___%

Full analysis begins on page 2.
```

**Do:**
- Write the thesis before building the model — the thesis constrains the model, not the other way around. If the model output doesn't match the thesis, either the thesis is wrong or the model needs revision.
- State the downside scenario explicitly on page 1 — the IC should see the worst-case return before they read the base case rationale.
- Limit the executive summary to one page; every number on page 1 must appear with full detail later in the memo.

**Don't:**
- Open with the property address, year built, or brokerage relationship — that information belongs in the property-description section, not the executive summary.
- Present only the base-case IRR without the downside; an IC memo with a single return figure is incomplete.
- Write the thesis after the analysis to justify the numbers — a thesis written to match the model is a rationalization, not an investment decision.

## Edge cases / when the rule does NOT apply

Preliminary deal screen memos (initial sourcing evaluation before full underwriting) may use a shorter format — thesis + estimated returns + top-3 risks — without the full IC memo structure. The one-page summary rule still applies.
